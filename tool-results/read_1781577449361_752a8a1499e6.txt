  1283→> **v3.14 — consecutive figure overlap fix (43 → 0 body-text overlaps).**
  1284→>
  1285→> **Root cause analysis:** `\noindent` in `\swarmwrapnext` does NOT force a
  1286→> paragraph break when called mid-paragraph (LaTeX's `\noindent` just
  1287→> suppresses indent; it doesn't call `\par`). When consecutive swarmwrap
  1288→> blocks have no blank lines between them, ALL text merged into ONE
  1289→> paragraph. Only the FIRST figure's parshape was applied — subsequent
  1290→> figures' `\parshape` was silently queued for the next paragraph (which
  1291→> never came). This caused 42 FULL_WIDTH_THROUGH body-text overlaps on
  1292→> the 50-figure stress test. Also explained the suspicious constant
  1293→> `remaining=261pt` — `\pagetotal` was never updated between swarmwrap
  1294→> calls because the paragraph never ended.

  1297→> (1) Add `\par` at the start of `\swarmwrapnext`. In vertical mode this
  1298→>     is a no-op. In horizontal mode it forces the previous paragraph to
  1299→>     end, ensuring each figure gets its own paragraph boundary. This also
  1300→>     fixes `\pagetotal` accuracy — remaining values now vary correctly
  1301→>     per figure instead of being stuck at 261pt.
  1302→> (2) Track figure height remaining after paragraph via `post_linebreak_filter`.
  1303→>     After TeX breaks the paragraph into lines, the callback counts total
  1304→>     hlist nodes vs `nl`. If `total_lines < nl`, the figure extends beyond
  1305→>     the paragraph. The excess `(nl - total_lines) * baselineskip` is stored
  1306→>     in `\swarmwrap@remaining`. The next `\swarmwrapnext` injects
  1307→>     `\vspace{\swarmwrap@remaining}` to push subsequent content below the
  1308→>     previous figure.
