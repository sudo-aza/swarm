### Root Cause Analysis

The Programmer removed explicit `\parshape \swarmwrap@pshape\relax`
from both code paths, relying entirely on the everypar mechanism.
However, everypar fires at paragraph boundaries — when TeX starts
a new paragraph, it executes the everypar token list which applies
`\parshape`. But the FIRST paragraph after `\swarmwrapnext` starts
BEFORE everypar fires, so it runs at full width with no parshape
and overlaps the figure.

The v3.54 approach (explicit `\parshape` + everypar for subsequent
paragraphs) correctly wrapped the first paragraph, but the explicit
parshape persisted across page breaks causing the leak regression.
The v3.55 approach (everypar only) eliminates leaks but breaks
wrapping entirely.

**The correct fix must do BOTH:** apply parshape to the first
paragraph AND make it clearable at page boundaries.

#### Root Cause Analysis

Traced the code flow in swarmwrap.sty:

1. **Figure page buildpage_filter** (line 819): When `rnl > 0 and
   tw_lua > 0`, sets `swarmwrap_leak_tw = tw_lua` (the narrow text

   `swarmwrap_leak_tw <= 0` — this is `true` because it was cleared
   in step 2. Returns early. The widening code (lines 847-858) NEVER
   executes.

The widening loop at lines 847-858 is dead code — it can never
execute because leak_tw is always cleared before any non-figure
page ships.

#### Proposed Fix

Remove `swarmwrap_leak_tw = 0` from the `page_had_fig` branch
(line 835). The early return (line 836) is still needed — it
prevents the filter from widening narrow lines that LEGITIMATELY
wrap around figures on the figure page. But leak_tw must NOT be
cleared, so it persists for the leak page's pre_shipout_filter.

The next figure page's buildpage_filter will overwrite leak_tw
with the new value, preventing stale cross-figure interference.
Non-figure, non-leak pages between figures would iterate the
widening loop, find no matching narrow lines, and clear leak_tw
harmlessly at line 860.

#### Why the post_linebreak_filter ghost_widen Also Fails

continuation pages. Possible reasons:
- An intermediate paragraph on the figure page consumes ghost_widen
- TeX processes the continuation lines in a separate callback

- The node list passed to the continuation invocation doesn't
  contain the expected hbox nodes

pre_shipout_filter is the primary fix.
