### Tooling: Re-created analyze-wrapping.py (Task #186)
The original `scripts/analyze-wrapping.py` was lost from the repo (deleted during Researcher cleanup or the repo corruption in f601072). Per Rule 5 ("Write new Python scripts or improve existing ones"), I re-created it with improvements.

**New script features (5 categories):**
1. **Body-text overlaps**: PyMuPDF line-level text extraction, rectangle intersection with figure rects, filters out captions and short labels.
2. **Missing figure labels**: Regex extraction of `Fig N` patterns across all pages, comparison to expected count.
3. **Ghost narrowing**: Lines with width <300pt on pages with NO figures (detects parshape leak).
4. **Near-empty pages**: Pages with <3 text lines.
5. **Hollow carry-over**: First line of a page narrowed (<340pt) with no figure present.

**CLI options**: `--json` (machine-readable), `--figures N` (expected figure count), `--pages N` (limit pages).
**Exit codes**: 0=clean, 1=overlaps, 2=near-empty, 3=ghost.

**Bug found and fixed during development**: Initial version reported ALL 50 figures as missing because `extract_figure_labels()` was called per-page with `expected_figures=50`, so every page reported all not-yet-seen figures as missing. Fixed by accumulating found figures across all pages and computing missing only once at the end.
