### pbv Parshape Leak Regression Analysis
The figure stack pop in post_linebreak_filter can RESTORE remaining@nl and everypar after they were consumed. If a subsequent paragraph spans a page break after this restoration, the narrow parshape carries to the next page. The pre_shipout_filter clears state on page ship, but lines already formatted before the ship retain their narrow shape.

Hypothesis: in pbv's specific test patterns (each scenario = 1 figure + 1 lipsum paragraph + \newpage), the stack pop restores state from a previous figure that's no longer visually present, causing narrow text to leak across the \newpage boundary. The stress-50 test avoids this because figures are densely packed (multiple per page) and the stack is always populated with current-page figures.

### NEW FINDING: Overlap Detection Methodology
Block-level detection (used in T89) is unreliable for parshape-wrapped documents. Text block bounding boxes span the full page width even when individual lines are narrowed. T89's "4 body-text overlaps" were likely false positives — line-level detection finds 0 in v3.39 with the same test. **Recommendation:** Future QA turns should use LINE-LEVEL text detection exclusively.

### Corrected Understanding
1. **T93's "28→40 regression" was FALSE.** The 40 leaks are pre-existing.
2. **T89's baseline of 28 was likely incorrect.** T89 compiled with a stale v3.36 working tree (broken git index), so the 28-leak count may have been for a different .sty version or measured in a different way.
3. **The no-stack version produces the same byte count as T89's v3.37 baseline** (45170b), confirming it's essentially v3.37 behavior.

### Confirmed Root Cause
TeX's paragraph breaker applies narrow parshape to the ENTIRE paragraph atomically. When a paragraph starts narrow (next to a figure) and TeX runs out of vertical space, it breaks the paragraph across pages — but the overflow lines retain the narrow parshape. The `pre_shipout_filter` clears everypar AFTER the page ships, but the damage is already done: TeX has already formatted the overflow lines with the narrow shape.

**Evidence:** Page 1's last narrow line ends with "Pel-" (hyphenated "Pellentesque"). Page 2's first leaked line starts with "lentesque habitant..." — a continuation of the SAME paragraph. This confirms TeX broke the paragraph across pages within the narrow zone.

### Why Stress-50 Is Unaffected
Stress-50 has 2-4 figures per page. Every page has a figure. The detect-parshape-leak.py script only flags figure-LESS pages as leaks. Narrow text on a page WITH a figure is expected wrapping behavior. So even if the same cross-page break occurs in stress-50, the leaked narrow text ends up on a page that also has a figure, and the script doesn't flag it.

### Root Cause of Failed Fix
The Programmer's guard checks: `remaining - used < fig_h` where:
- `remaining` = `tex.dimen[0] / 65536.0` (TeX's view of remaining space on the page)
- `used` = `swarmwrap_page_fig_height` (accumulated actual figure heights)
- `fig_h` = new figure's height

The problem: `tex.dimen[0]` is TeX's REMAINING space, which is calculated as `pagegoal - pagetotal`. Because `\smash{\rlap{...}}` makes figures zero-height in TeX's accounting, `pagetotal` does NOT include the physical space occupied by smashed figures. So `remaining` is INFLATED — it includes space that smashed figures already physically occupy.

When the guard evaluates `remaining - used < fig_h`:
- `remaining` = (large, because TeX doesn't know about smashed figures)
- `used` = (correct accumulated figure heights)
- Result: `remaining - used` is still large enough that the condition is FALSE, so `deferred` is NOT set, and the figure is placed and clips.

**In essence: the guard subtracts `used` from an already-inflated `remaining`, which negates the correction. The fix would only work if `remaining` reflected the ACTUAL physical remaining space, not TeX's incorrect view.**

### Suggested Fix Approach (for Task #193)
Instead of using TeX's `remaining` (which is wrong), track the actual Y coordinate:
- Maintain `swarmwrap_last_fig_bottom_y` (reset to `top_margin` on page ship)
- After placing each figure, update: `swarmwrap_last_fig_bottom_y = swarmwrap_last_fig_bottom_y + fig_h`
- Before placing, check: `swarmwrap_last_fig_bottom_y + fig_h > page_height - bottom_margin`
- If TRUE, set deferred=1 (force page break before this figure)
