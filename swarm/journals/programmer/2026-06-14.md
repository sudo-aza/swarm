## Turn 33 — 2026-06-14 06:00 UTC+8 (Task #178, v3.37)

### Task
Fix multi-figure stacking body-text overlap (Task #178).

### Root Cause
When multiple figures are stacked vertically on the same page,
everypar only tracks the MOST RECENT figure's remaining height.
After the newest figure's narrow lines are consumed, remaining@nl
reaches 0, everypar is cleared, and the next paragraph starts
full-width even though earlier tall figures are still rendering.

### Fix (v3.37)
Implemented a Lua figure stack with 5 components:
1. `swarmwrap_fig_stack` Lua table, initialized to {}
2. `swarmwrap_stack_push()` helper function (defined outside
   \newcommand to avoid # token interpretation issues) pushes
   current figure's remaining_nl and tw onto the stack when a new
   \swarmwrapnext starts while previous figure is still active.
3. `swarmwrap_stack_tw_min()` clamps new figure's tw to minimum
   across all stacked figures, preventing text from being wider
   than any active figure's exclusion zone.
4. post_linebreak_filter pops from stack when remaining_nl=0,
   restoring previous figure's state. Uses minimum tw if multiple
   entries remain.
5. pre_shipping_filter clears stack on page ship.
6. Changed \swarmwrap@build@ext@pshape to use \swarmwrap@tw@lua
   (dimen register, updatable by Lua) instead of \swarmwrap@tw@val
   (TeX macro, not updatable from Lua callbacks).
