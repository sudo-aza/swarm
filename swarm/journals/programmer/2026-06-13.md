## Turn 11 (06:00 UTC+8, cron)

**Task:** #205 — Fix missing parshape line count

**Root Cause (QA T65 confirmed):**
`\swarmwrap@pshape` was built starting with `\def\swarmwrap@pshape{}`
(empty), then appending indent/width pairs via the loop. This produced
specifications like `0pt 232.19pt 0pt 232.19pt ...` with NO leading
line count. TeX's `\parshape` primitive reads the first integer as the
count — in this case `0` (from `0pt`), meaning 0 shaped lines (all
full-width). The remaining specification text leaked as visible text
on PDF pages.

The count WAS prepended (lines 1241-1247) but ONLY inside the
`\ifdim\swarmwrap@tw@lua<\swarmwrap@saved@origtw` conditional — which
is FALSE for the first figure and whenever tw wasn't clamped.

**Fix (v3.54):**
One-line change: `\def\swarmwrap@pshape{}` →
`\xdef\swarmwrap@pshape{\the\numexpr\swarmwrap@nl+1\relax\space}`

This ensures the line count is ALWAYS present at the start of the
parshape specification, regardless of code path. The rebuild inside
the `\ifdim` block remains as a no-op fallback for the clamp case.
