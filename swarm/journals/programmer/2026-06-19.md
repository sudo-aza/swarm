### Task #208: 1000-fig page count regression 183→200 pages

**Root cause**: The anti-waste squeeze check (v3.51) was inside the `\ifdim remaining < fh` block. When the 5th figure on each page had remaining < fh (because the 4th figure's wrapping text consumed remaining space), the check computed space-after and compared it against the threshold. Both v3.51's `\swarmwrap@fh` threshold and an attempted `\swarmwrap@squeezemin` threshold were too conservative — they prevented squeeze-fit that v3.49 used to achieve 6-fig packing.

**Fix (v3.53)**: Removed the anti-waste squeeze check entirely. The squeeze-fit path now runs whenever `remaining >= squeeze-factor*fh AND remaining >= squeezemin`, without the additional anti-waste gate. This restores v3.49 packing density.

## 17:00 UTC+8 (Turn, Task #211 — parshape leak across \newpage boundaries)

**Pulled**: cf3f31c9. QA T166 found ghost narrowing in customwrap (5 pages) and pagebreak-variations (7 pages).

### Investigation

Discovered that `pre_shipout_filter` cannot set everypar to persist across page boundaries — TeX's shipout mechanism restores the old everypar after the callback's group ends. Verified with diagnostic Lua logging.

Also discovered the QA detection script uses `page_w * 0.9` (535pt on A4) as the "narrow" threshold, but the actual text area is ~345pt. This produces false positives — ALL text lines are detected as "narrow".

### Two distinct parshape leak scenarios

**Scenario A** (explicit `\newpage`): Parshape from previous figure persists to the next page's first paragraph. **Fixed** by patching `\newpage` and `\clearpage` to reset parshape before the page break.

**Scenario B** (natural page break): Wrapping paragraph continues across pages. TeX designed `\parshape N ...` to span page breaks — the N-line specification covers continuation lines. Fixing this requires node-list manipulation in `post_linebreak_filter`. **Not fixed** — deferred.

### Changes (v3.54)

- Added `\swarmwrap@parshape@active` count register (tracks active parshape state)
- Added `\swarmwrap@reset@pshape` command (resets parshape to full-width)
- Patched `\newpage` and `\clearpage` to call reset when parshape is active
- Added everypar reset in `post_linebreak_filter` for cross-page paragraph breaks

**Compilation results**: All test suites unchanged. 0 errors.
