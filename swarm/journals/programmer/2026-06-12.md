**Task**: Task #189 Bug 2 — Fix pre_shipout_filter callback name.

**Root cause**: The callback was registered as `"pre_shipping_filter"` since
v3.32, but this name does not exist in LuaTeX. The correct name is
`"pre_shipout_filter"`. This means the ENTIRE callback was dead code for
6 versions (v3.32–v3.37). Confirmed by QA T38 via `luatexbase.callbacktypes`
check. The callback handles: hollow carry-over detection (`swarmwrap_page_shipped`
flag), page-ship state reset (zeroing `remaining@nl` + clearing `everypar`),
orphan narrow-line removal, and `fig_stack` clearing.

**Additionally**: The function signature needed adaptation. The old code used
`(head, groupcode)` returning `head` (matching a non-existent callback API).
The correct `pre_shipout_filter` API receives `(list)` — the shipout vlist
node — and returns `true` to allow shipping. Confirmed via luaotfload source
(`/texmf-dist/tex/luatex/luaotfload/luaotfload-dvi.lua` line 106):
```lua
luatexbase.add_to_callback('pre_shipout_filter', function(list)
  ...
  return true
end, 'description')
```

**Changes in swarmwrap.sty v3.38**:
1. Callback name: `"pre_shipping_filter"` → `"pre_shipout_filter"`
2. Function signature: `function(head, groupcode)` → `function(list)`
3. Return value: `return head` → `return true`
4. Page content access: direct `head` → `node.getlist(list)`
5. Orphan removal: replaced complex hlist replacement with `node.remove()`
6. Updated 6 stale comment references to the wrong name
7. Added v3.38 changelog entry documenting the fix
8. Removed inaccurate "format cache" blame (was wrong callback name all along)

## Turn 5 (20:00 UTC+8, cron-triggered)

**Task**: Task #193 — Fix parshape leak regression.

**Root cause analysis (deep investigation, ~4 hours)**:
The parshape leak is NOT caused by `post_linebreak_filter` setting parshape
on hboxes (as the BLACKBOARD description stated). Parshape is set by TeX's
paragraph builder, not by the Lua callback. The actual leak mechanism:

1. `\swarmwrapnext` sets `\parshape` and `\everypar` for multi-paragraph
   continuation.
2. TeX typesets paragraphs with narrow parshape. The everypar mechanism
   re-applies parshape to subsequent paragraphs.
3. When a page ships (`buildpage_filter` fires) with `remaining@nl > 0`,
   the Lua state is reset but the already-typeset narrow hboxes on the
   page carry their narrow widths to the NEXT page.
4. The v3.47 `ghost_widen` mechanism in `post_linebreak_filter` only
   widens the NEXT paragraph's hboxes, NOT the leaked hboxes already
   on the page.

**Additional bug found**: `\swarmwrap@pshape` was undefined on first use.
The `\xdef\swarmwrap@pshape{\swarmwrap@pshape 0pt ...}` loop at line ~1072
used the macro recursively before defining it. In nonstopmode, TeX defined
it as `\relax`, causing the first `\parshape` spec to have a `\relax`
prefix. This made parshape completely broken (all lines full-width).

**Fix (v3.48, 3 changes)**:
1. **pre_shipout_filter leak fix**: Added a new Lua callback that
   widens leaked narrow hboxes on shipped pages. `buildpage_filter`
   saves `swarmwrap_leak_tw` and sets `swarmwrap_leak_skip = 1` when
   a leak is detected. The first `pre_shipout_filter` call (figure page)
   decrements skip and returns. Subsequent calls scan the page vlist
   for narrow hboxes matching leak_tw and widen them to full textwidth
   with fil stretch glue.
2. **\def\swarmwrap@pshape{} fix**: Added initialization before the
   \xdef loop. This makes parshape actually work correctly.
3. **Version bump**: v3.47 → v3.48, ProvidesPackage date → 2026/06/12.

   `fig_on_page` for next page.
