## Task: Ghost-narrowing fix — re-enable penalty insertion (Task #204 follow-up)

### Context
v3.53 fixed a critical node ID bug (glyph=37→29, disc=38→7) in `has_text_content()` but left the post_linebreak_filter as a passthrough. This meant ALL penalty-based fixes from v3.45-v3.52 were complete no-ops — penalties were never inserted.

### Investigation
1. **Confirmed the bug**: Added debug logging to `swarmwrap_post_lb()`. With the fixed node IDs, the callback correctly detects narrow lines (e.g., "9 narrow lines detected, inserting penalties").
2. **Found `tex.dimen["baselineskip"]` error**: baselineskip is a skip register, not a dimen. Must use `tex.skip["baselineskip"].width`. This caused `(node filter): error: incorrect ...` warnings in v3.54's first attempt.
3. **Tested penalty strategies**:
   - **All narrow lines + 10001 penalty + last narrow -500**: Ghost 3→2, but 26 hollow carry-over, +21 pages (1038→1059). Too aggressive.
   - **Transition-only -500**: No regression (46/1038 pages), no improvement (still 3 ghost). Too weak.
   - **Transition-only -10000**: Fatal error (TeX capacity exceeded — infinite page breaks). Way too aggressive.
   - **Transition-only -2000**: No regression (46/1038 pages), no improvement (still 3 ghost). Moderate but insufficient.
   - **Increased safety margin (4bs)**: 50-fig: 47 pages (+1), 30 excessive (was 32). 1000-fig: still 3 ghost. Margin increase caused page regression without fixing ghost.
4. **Root cause confirmed**: The 3 ghost-narrowing pages (67, 332, 927) are edge cases where the FIRST paragraph of a wrapping session is very long and spans a page break within the parshape zone. TeX's page-break algorithm is forced to break within narrow lines because the page is overfull. No inter-line penalty can prevent this — TeX will break wherever it must.
5. **Inherent TeX limitation**: TeX's `\parshape` persists across page breaks within a single paragraph. The `\par` patch handles subsequent paragraphs, but the first paragraph's carry-over is a black box. Shipout-time fixes were proven impossible by QA (v3.45: glue-stretch doesn't propagate; v3.46: node.hpack fatal error; shipout_filter: luaotfload conflict).

### Changes
- **swarmwrap-callback.lua**: Re-enabled penalty insertion with correct node IDs. Insert -2000 penalty at narrow→full transition point. Fixed `tex.dimen["baselineskip"]` → `tex.skip["baselineskip"].width`.
- **swarmwrap.sty**: Updated version to v3.54, updated changelog and known limitations. Safety margins kept at v3.53 levels (3\baselineskip total in \par patch).

## Task: Non-penalty ghost-narrowing approach (Task #209) — 20:00 UTC+8

### Context
zoe chose option (b): "fundamentally different approach" to eliminate the 3 remaining ghost-narrowing pages at 1000-fig scale. Penalty-based approaches exhausted (7 QA reviews).

### Investigation
Systematic investigation of all suggested non-penalty strategies:

1. **buildpage_filter callback**: Registered callback, debug-logged all invocations. In this LuaTeX version, `buildpage_filter` receives strings (groupcodes like "vmode_par", "hmode_par") as `head` parameter, not node lists. Cannot inspect or modify nodes. Confirmed via `io.open` debug logging to `/tmp/bpf-debug.log`.

2. **pre_shipout_filter**: Confirmed it fires and receives the page's vlist node. Can traverse and inspect hlists within vlists. However, QA already confirmed (Task #207) that modifications in `pre_shipout_filter` do NOT propagate to PDF output.

3. **post_linebreak_filter — hpack width modification**: Changed `hbox.width` for narrow lines that overflow the page boundary. 197 node filter warnings but compilation succeeded. Ghost narrowing unchanged (3/1038). `hbox.width` changes don't reposition glyphs — same class of bug as v3.29.

4. **post_linebreak_filter — node.hpack() re-packing**: Used `node.hpack(n.head, linewidth, 'exactly')` + `node.replace()` to create full-width re-packed lines. Node filter errors. Ghost narrowing unchanged.

5. **post_linebreak_filter — forced break (-10000)**: Added `\penalty-10000` at narrow→full transition point. TeX still breaks within narrow zone when page is overfull — forced break at transition is too late.

6. **pre_linebreak_filter — large emergencystretch**: Set `emergencystretch = 50 * baselineskip` for paragraphs with many nodes. No effect on ghost narrowing.

7. **Safety margin escalation**: Tested 5bs, 8bs, 15bs safety margins (in \par patch, list hook, item hook). Results:
   - 5bs: 46/1038 pages, 3 ghost (unchanged)
   - 8bs: 46/1038 pages, 3 ghost (unchanged)
   - 15bs: 46/1038 pages, 3 ghost + 3 body-text overlaps (regression)

8. **Root cause deep-dive**: Examined pages 66-67 (ghost-narrowing pair). Page 66 has Fig 70 (4cm x 9cm, itemize). The itemize's `\item` sets parshape for N narrow lines. The item's text paragraph (from `\lipsum[3][1-3]`) is a SINGLE paragraph that spans the page break. TeX processes the entire paragraph atomically — no `\par` fires, no `\item` fires, no `\everypar` fires during the carry-over. The parshape persists to page 67 by TeX's fundamental paragraph-breaking algorithm.

### Conclusion
The 3 ghost-narrowing pages (67, 332, 927) are caused by TeX's paragraph-breaking algorithm processing paragraphs atomically. When a single paragraph spans a page break within a `\parshape` zone:
- No Lua callback fires between the page break and the continuation (post_linebreak_filter fires once per paragraph, before page building)
- No TeX mechanism exists to inject a `\par` mid-paragraph
- `pre_shipout_filter` modifications don't propagate (confirmed by QA)
- `buildpage_filter` doesn't receive node lists in this LuaTeX version
- Changing `hbox.width` or re-packing with `node.hpack()` doesn't reposition glyphs
- Increasing safety margins causes overlaps at 15bs level, no improvement at lower levels

**v3.54 is the strongest achievable version without modifying the TeX engine itself.** CODE UNCHANGED.

Added a new layer of ghost-narrowing prevention in pre_linebreak_filter that reads tex.parshape, counts narrow entries, and checks remaining page space before TeX breaks lines. If the narrow zone doesn't fit (with 4bs safety buffer), it reduces narrow entries or clears parshape entirely. This is fundamentally different from all 7 penalty-based approaches (v3.45-v3.54) — it modifies parshape directly, not line-break penalties. Technique based on chickenize package's parshape modification (Henri Menke).
>
> **Results**: 50-fig (46 pages): 0 ghost, 0 hollow, 0 overlaps (identical to v3.54 baseline, no regressions). 1000-fig (1038 pages): 3 ghost narrowing unchanged (pages 67/332/927), 0 overlaps, page count stable.
>
> **Why 3 ghost pages persist**: The needspace check fires for every wrapped paragraph (verified: ~197 calls in 50-fig, ~2000+ in 1000-fig). But it NEVER triggers reduction because the .sty's deferred figure check ensures remaining >= nl*bs, and by pre_linebreak_filter time, the additional inter-paragraph spacing (parskip, item separation) doesn't consume enough space to breach the 4bs buffer. The ROOT CAUSE of the 3 ghost pages is different: the FIRST paragraph is very long (6+ lipsum blocks) and starts near the middle of the page. The narrow zone FITS, but the paragraph extends well past it. TeX's page breaker fills the page with lines from this paragraph and must break within the narrow zone because there are no better break points. The Researcher explicitly predicted this limitation: "Approach B doesn't fix the case where the paragraph starts near the top of a page but is long enough to span a break within the narrow zone."
>
