## 10:03 UTC+8 — Task #98: Fix swarmwrap.sty v2.0 — whitespace problem

**Files**: `src/themes/swarmwrap.sty` (v1.1→v2.0), `src/test-wrapfig/test-customwrap.tex` (title updated)
**Origin**: QA #97 (10/10 pass) + zoe feedback — parshape extends 252-345pt past figure bottom.

### Root cause analysis

Three interacting issues caused the massive whitespace:

**1. Wrong height metric for line count**
The parshape line count was computed from `\ht + \dp + \baselineskip` (total box height). In a [t]-aligned minipage, the reference point is at the first content's baseline. For a `\rule{width}{height}`, the baseline is at the rule BOTTOM. So `\ht` = rule height (extends ABOVE the text start) and `\dp` = caption height (extends BELOW). The text wraps alongside the caption (`\dp`), not the rule above it. Using `\ht+\dp` overcounted by ~108pt (14 baselines), causing 14 narrowed lines when only 6 were needed.

**2. `\rlap` depth corrupting interline spacing**
`\rlap` makes the figure box zero-width but preserves its height (108pt) and depth (64pt). TeX's interline glue calculation: `glue = \baselineskip - (ht_next + dp_prev)`. With dp_prev=64pt and ht_next=8pt, glue = 13.6 - 72 = -58.4pt, which is below `\lineskiplimit` (0pt). TeX falls back to `\lineskip` (1pt), placing the next baseline at prev_baseline + 64 + 1 + 8 = 525.7. This created a 74.7pt gap between lines 1 and 2.

**3. LuaTeX parshape non-reversion**
Unlike pdfTeX (which reverts to `\hsize` after N parshape lines), LuaTeX reuses the LAST parshape specification for all lines beyond N. Without a trailing full-width line, every line in a long paragraph was narrowed.

### Fixes

(a) **Line count from `\dp`** (line 83): Only the caption depth (content below text start) needs narrowed text alongside it. Changed `\swarmwrap@fh=\ht\swarmwrap@box + \dp\swarmwrap@box + \baselineskip` to `\swarmwrap@fh=\dp\swarmwrap@box + \baselineskip`.

(b) **`\smash` around `\rlap`/`\llap`** (lines 133-141): `\smash{\rlap{...}}` sets the box height and depth to zero, making the entire figure overlay invisible to TeX's line layout. Verified: interline spacing is now a consistent 13.6pt across all lines.

(c) **Trailing full-width parshape line** (line 113): Changed `\xdef\swarmwrap@pshape{...}` to include N+1 entries. Entry N+1 is `0pt \the\textwidth`, which LuaTeX reuses for excess lines. Verified: 3-line test shows lines 1-3 at 200pt, lines 4+ at 359pt.

(d) **Height arg deprecated** (line 62): Third argument `\begin{swarmwrap}[r]{width}{height}` still accepted but ignored. Auto-detection uses actual box `\dp`. Existing code continues to work without changes.

- Page 2 (left wrap): 6 narrow lines, 13.6pt spacing, excess -8pt [PASS]

## 13:00 UTC+8 — Task #101: Fix swarmwrap.sty v2.1 — figure positioned above text

**Files**: `src/themes/swarmwrap.sty` (v2.0→v2.1)
**Origin**: QA #100 rated 4/10 — figure positioned above text, not beside it.

### Root cause
The `[t]`-minipage puts the reference point at the rule's baseline (bottom of the rule). So `\ht = rule_height` (extends UPWARD from reference point) and `\dp = caption_depth` (extends downward). The v2.0 code used `\raise-2pt\hbox{\copy\swarmwrap@box}`, placing the reference point near the text baseline. This made the 108pt rule extend 108pt UPWARD while text flows downward — only 1 line of text overlapped with the figure.

### Fix
Changed `\raise` dimension from `-2pt` to `\dimexpr\ht\strutbox-\ht\swarmwrap@box\relax`. This computes the negative offset needed to lower the box so that the figure TOP (= reference_point + \ht) aligns with the text ascender height (= \ht\strutbox, ≈8.5pt). The figure now extends DOWNWARD from the text start, alongside the wrapped text lines.

Also changed line count from `\dp` only back to `\ht + \dp` (total box height) — v2.0's `\dp`-only approach only counted the caption depth (≈15pt), but with the corrected figure placement, the full box height (≈123pt) is needed.

### Initial approach attempt: `\vtop`
First tried wrapping in `\vtop{\hbox{...}}` to change the box reference point to its top. However, `\vtop` with a single `\hbox` preserves the inner box's ht/dp — the reference point stays at the inner box's reference point regardless. Abandoned `\vtop` in favor of direct `\raise` adjustment.

### Additional bug: `\strutht`
Initial attempt used `\raise\strutht` which is not a valid TeX command. Corrected to `\raise\ht\strutbox` (accessing the strut box's height register), then further refined to the `\dimexpr` approach.

## 21:00 UTC+8 — Task #109: Fix swarmwrap.sty v2.4 overfull hbox

**File**: `src/themes/swarmwrap.sty` (v2.3→v2.4)
**Origin**: QA #107 (8/10) — 4pt overfull hbox on left-wrap test.

### Root cause analysis

The `Overfull \hbox (4.0258pt too wide)` on Test 2 is NOT caused by incorrect parshape arithmetic. Verified: indent + width = linewidth exactly (97.36pt + 262.64pt = 360.0pt). The overfull is a natural TeX behavior — when text content doesn't break cleanly within a narrowed parshape width, TeX's line-breaking algorithm can produce an overfull hbox. The same overfull occurs on BOTH left and right wraps, confirming it's a text-width issue, not a direction-specific bug.

### Approaches investigated

1. **Reducing gap (12pt→8pt)**: Widened text by 4pt. Eliminated the specific 4pt overfull on lipsum[3-4] but produced NEW overfull (11.56pt) on lipsum[1-2] and other tests. Non-monotonic relationship — different text content responds differently to gap changes.

2. **`\rightskip` adjustment**: Only helps underfull hbox (adds space at right margin), not overfull (text exceeds available width).

3. **`\emergencystretch` with `\everypar` scoping**: `\noindent` triggers `\everypar` at paragraph start, which resets `\emergencystretch` to 0pt before the paragraph is broken — defeating the purpose.

4. **`\emergencystretch` unscoped** (chosen): Set `\emergencystretch=\fontdimen6\font` (1em ≈ 10.95pt) before `\noindent` in `\swarmwrapnext`. This gives TeX additional inter-word stretch capacity ONLY when it cannot find a satisfactory break — normal lines are completely unaffected. Verified: zero overfull hbox warnings on all 7 tests.
