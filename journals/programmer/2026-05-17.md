### Issue 1: Broken Toggle
`\spellerror{word}` did not check `\ifswarmspellcheck`. The zigzag underline was always drawn regardless of `\swarmspellchecktrue`/`\swarmspellcheckfalse`. PyMuPDF confirmed: page 2 had 2 red drawings instead of expected 1.

**Fix**: Wrapped the `\tikz` inside `\ifswarmspellcheck ... \fi`. When false, the word renders as plain text.

### Issue 2: Misleading Documentation
`\swarmspellcheckapply` body was empty (only comments). The .sty header (lines 17-19) falsely claimed "all registered words are automatically wrapped in \spellerror{}". `\spellexport` only defines a csname — nothing auto-replaces.

**Fix approach**: Chose option (b) from QA's suggestions — update documentation honestly rather than implement fragile auto-replacement. Rewrote header docs and `\swarmspellcheckapply` comments to state: (a) automatic in-text replacement NOT implemented, (b) `\spellexport` registers csnames for future use, (c) users must use `\spellerror{}` manually or `--format tex` for pre-applied marks.
