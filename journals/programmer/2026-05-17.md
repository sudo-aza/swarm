## 01:00 UTC+8 — Task #89: Pure-Lua spellcheck research and benchmark

**Files**: `notes/task-89-lua-spellcheck-research.md` (new), `BLACKBOARD.md` (tasks #91-#94 added)
**Origin**: HIGH PRIORITY request from zoe — investigate if Lua spellcheck would be faster on Raspberry Pi.

### Research
- No existing Lua spellcheck library on LuaRocks or CTAN
- Only viable approach: custom Lua hash table from plain-text word list
- pyspellchecker's English dictionary: 160,572 words, 1.5 MB plain text

### Benchmarks

| Metric | Python (two-pass) | Lua (single-pass) |
|--------|-------------------|-------------------|
| Spellcheck overhead | 0.19-0.30s | 0.08s (dict load) |
| Memory | ~30 MB (process) | +14-20 MB (hash table) |
| Engine support | pdfLaTeX + XeLaTeX + LuaLaTeX | **LuaLaTeX only** |
| CI output | JSON + terminal | **None** |
| Suggestions | Yes | **No** |

### Decision: Keep Python approach

The ~0.1-0.2s speed savings (3-6% of total compilation time) do not justify:
1. Losing pdfLaTeX/XeLaTeX compatibility
2. Losing CI output formats
3. Losing spelling suggestions
4. Adding 14-20 MB memory overhead
5. Callback complexity and interaction bugs

Alternative optimizations for Raspberry Pi: cache helper files, lightweight dictionary mode, daemon mode.

## 05:00 UTC+8 — Task #90: swarmwrap.sty v1.0 — Custom float wrapper

**Files**: `src/themes/swarmwrap.sty` (new, 115 lines), `src/test-wrapfig/test-customwrap.tex` (new, 141 lines)
**Origin**: GATEKEEPER task #60 activated — all 9 tested packages failed QA.

### Architecture
`\begin{swarmwrap}[r]{width}{height}...\end{swarmwrap}\swarmwrapnext` pattern:
1. Environment saves figure content into `\savebox`
2. Computes parshape spec: text width (textwidth - figwidth - gap) and line count
3. Stores all values globally via `\xdef` (needed because environment creates TeX group)
4. `\swarmwrapnext` applies `\parshape` to following paragraph, overlays figure via `\rlap`/`\llap`

### Key bugs fixed during development
1. `\toks` register stored unexpanded tokens — `\parshape` couldn't read numbers. Fixed with `\xdef`.
2. Local `\edef` lost when environment group closes. Fixed with global `\xdef`.
3. `\ifx` compared macro token vs character (always false for `[l]`/`[r]`). Fixed with helper macros.
4. Boolean and dimen registers were local to group. Fixed with `\global` and `\xdef` macros.
5. `\rlap{\kern...\copy\box}` invisible in LuaLaTeX. Fixed with `\hskip`.

## 09:00 UTC+8 — Task #96: Fix swarmwrap.sty v1.1 — savebox, boolean, and positioning

**Files**: `src/themes/swarmwrap.sty` (v1.0→v1.1), `src/test-wrapfig/test-customwrap.tex`
**Origin**: QA #95 rated 3/10 — figures not rendered in PDF.

### Bug 1: Savebox content lost on group exit (critical)
`\newsavebox` assigns are LOCAL to the TeX group. When `\end{swarmwrap}` closes the group, the savebox reverts to empty. The `\xdef` macros for parshape persisted (global), which is why text wrapping worked but figures were invisible.

**Fix**: Added `\global\setbox\swarmwrap@box=\box\swarmwrap@box` after `\end{lrbox}` (line 70). This copies the box content globally before the group closes.

### Bug 2: `\ifswarmwrap@right` always FALSE
`\swarmwrap@righttrue` in the begin code (line 50) was LOCAL — no `\global` prefix. When the environment group closed, it reverted to the default FALSE from `\newif`. This caused `\swarmwrapnext` to always take the left-wrap branch (`\llap`), placing figures at the left margin regardless of `[r]`/`[l]` option.

**Fix**: Changed `\swarmwrap@righttrue` to `\global\swarmwrap@righttrue` (line 50). This ensures the boolean persists after the environment group closes.

### Bug 3: Test 6 missing `\swarmwrapnext`
The multicol test had no `\swarmwrapnext` after the environment, so no wrapping was applied.

**Fix**: Added `\swarmwrapnext` after `\end{swarmwrap}` in Test 6 (line 140).

### Additional improvement
Rewrote `\swarmwrapnext` figure placement: wrapped `\copy\swarmwrap@box` in `\hbox{}` for more robust positioning. Left-wrap uses `\llap{\hskip\swarmwrap@ind@val...}` for correct indent-based placement.
