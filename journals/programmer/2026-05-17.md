## 00:00 UTC+8 — Task #88: Fix spellcheck.sty v1.1 toggle and auto-replacement

**File**: `src/themes/spellcheck.sty`
**QA issue**: QA #83 rated 8/10. Two issues found.

### Issue 1: Broken Toggle
`\spellerror{word}` did not check `\ifswarmspellcheck`. The zigzag underline was always drawn regardless of `\swarmspellchecktrue`/`\swarmspellcheckfalse`. PyMuPDF confirmed: page 2 had 2 red drawings instead of expected 1.

**Fix**: Wrapped the `\tikz` inside `\ifswarmspellcheck ... \fi`. When false, the word renders as plain text.

### Issue 2: Misleading Documentation
`\swarmspellcheckapply` body was empty (only comments). The .sty header (lines 17-19) falsely claimed "all registered words are automatically wrapped in \spellerror{}". `\spellexport` only defines a csname — nothing auto-replaces.

**Fix approach**: Chose option (b) from QA's suggestions — update documentation honestly rather than implement fragile auto-replacement. Rewrote header docs and `\swarmspellcheckapply` comments to state: (a) automatic in-text replacement NOT implemented, (b) `\spellexport` registers csnames for future use, (c) users must use `\spellerror{}` manually or `--format tex` for pre-applied marks.

### Changes
- `src/themes/spellcheck.sty`: v1.0 → v1.1
- `\spellerror{}` now checks `\ifswarmspellcheck` toggle
- Header documentation rewritten to accurately describe capabilities
- `\swarmspellcheckapply` comments updated (removed misleading "Scans all registered words")
- `\spellexport` docs clarified (registers csnames, does NOT auto-replace)

## 01:00 UTC+8 — Task #89: Pure-Lua spellcheck research and benchmark

**Files**: `notes/task-89-lua-spellcheck-research.md` (new), `BLACKBOARD.md` (tasks #91-#94 added)
**Origin**: HIGH PRIORITY request from zoe — investigate if Lua spellcheck would be faster on Raspberry Pi.

### Research
- No existing Lua spellcheck library on LuaRocks or CTAN
- Only viable approach: custom Lua hash table from plain-text word list
- pyspellchecker's English dictionary: 160,572 words, 1.5 MB plain text

### Benchmarks

| Metric | Python (two-pass) | Lua (single-pass) |
|--------|-------------------|-------------------|
| Spellcheck overhead | 0.19-0.30s | 0.08s (dict load) |
| Memory | ~30 MB (process) | +14-20 MB (hash table) |
| Engine support | pdfLaTeX + XeLaTeX + LuaLaTeX | **LuaLaTeX only** |
| CI output | JSON + terminal | **None** |
| Suggestions | Yes | **No** |

### Decision: Keep Python approach

The ~0.1-0.2s speed savings (3-6% of total compilation time) do not justify:
1. Losing pdfLaTeX/XeLaTeX compatibility
2. Losing CI output formats
3. Losing spelling suggestions
4. Adding 14-20 MB memory overhead
5. Callback complexity and interaction bugs

Alternative optimizations for Raspberry Pi: cache helper files, lightweight dictionary mode, daemon mode.
