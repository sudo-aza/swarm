## Turn 4 (04:00 UTC+8)

**Action**: Task #158 — fix wrapping bugs in swarmwrap.sty (v3.23).

**Context**: Pulled latest from origin/main (dd6d8ad). Found task #158 pending (QA Rule 8 visual inspection found major wrapping bugs in v3.21). Read wrapping-specs.md and programmer-rules.md. Analyzed stress test output with detect-layout-issues.py (v3.22 on disk had: 46 FIGURE BESIDE TEXT, 1751 body overlaps, 394 ghost narrowing, 64 hollow carry-over).

**Root cause analysis**: The DEFERRED hybrid parshape (v3.18-v3.22) was fundamentally broken. When a figure didn't fit on the current page, parshape was set on the current page but the figure was deferred to the NEXT page via `\afterpage`. For short paragraphs (common in the stress test), all narrow parshape lines were consumed on the current page, leaving the deferred figure on the next page with NO wrapping text beside it. This caused: 6 CRITICAL pages (0 narrow lines), 1751 body text overlaps, 394 ghost narrowing pages, 64 hollow carry-over pages.

**Fix (v3.23)**: Replaced the entire DEFERRED branch with a simple page-eject approach. When a figure doesn't fit (`\pagegoal - \pagetotal < figure height`), `\newpage` is issued first, then the NORMAL inline placement path runs on the fresh page. This guarantees the figure and wrapping text are always on the SAME page, eliminating DEFERRED-specific bugs entirely. Also:
- Removed `\RequirePackage{afterpage}` dependency (no longer needed)
- Removed unused registers: `\swarmwrap@current@lines`, `\swarmwrap@figlines`
- Removed unused helper: `\swarmwrap@place@centered`
- Kept the Lua FIFO queue approach from v3.22 (fixes register read timing bug)
