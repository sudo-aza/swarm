**Finding**: All 56 overlaps are from consecutive figures WITHOUT section breaks — NOT from itemize. The root cause is that LuaLaTeX's \everypar is NOT re-entrant (fires once per paragraph, consumed by LaTeX2e). The v3.18 everypar restart trick only applies parshape to the first paragraph.

### 11:00 UTC+8 — Fix deferred path: replace broken \afterpage with page eject (v3.23)
**Task**: Self-tasked — fix the broken deferred path in swarmwrap.sty.
**Root cause**: The deferred branch of \swarmwrapnext used \afterpage{} to place figures on the next page, but the afterpage package was never loaded (\RequirePackage{afterpage} was missing). This caused "Undefined control sequence" errors on every deferred figure. Even if loaded, \afterpage is fundamentally broken for wrapping: it inserts content at the top of the next page, but continuation lines from a split paragraph are already broken by TeX at full width and cannot be re-broken with a new parshape.
**Fix (v3.23)**: Replace \afterpage with page eject. When figure doesn't fit on current page: (1) reset parshape to full width (1 line, 0pt \linewidth), (2) reset emergencystretch to 0, (3) clear swarmwrap state (active=false, remaining=0, restore orig@par, clear everypar, clear list@active), (4) \newpage, (5) re-establish wrapping on new page (parshape, remaining, active, par patch, emergencystretch, tw@lua), (6) place figure inline with \noindent + \smash{\rlap{...}}.

### 14:00 UTC+8 — Fix body-text overlaps: remove parshape trailing reset (v3.25)
**Task**: Addressed the 72 body-text overlaps on the 50-figure stress test (Task #166 root cause).
**Root cause**: The parshape specification ended with `0pt \linewidth` (a full-width reset line). When a paragraph had more lines than `nl` (the computed narrow line count = figure height / baselineskip), lines after `nl` reverted to full width while the figure (placed via `\smash{\rlap{...}}`) was still physically present on the page. This caused text to flow through the figure area.
**Fix (v3.25)**: Removed the trailing `0pt \linewidth` reset line from ALL four parshape construction sites: (1) main `\swarmwrapnext` path, (2) `\par` patch for multi-paragraph extension, (3) list everypar hook, (4) item apply parshape hook. Also changed the parshape line count from `N+1` to `N`. Without the reset, TeX reuses the last parshape entry (narrow width) for all subsequent lines in the paragraph. The `\par` patch handles deactivation at paragraph boundaries by checking `\pagetotal >= fig_bottom`.
**Trade-off**: Text past the figure within the same paragraph remains narrow (cosmetic only, no overlap). Stress test page count increased from 37 to 50 due to more narrow text.
**Test results**:
- test-customwrap.tex: 8 pages, 0 errors, 0 body-text overlaps, 0 ghost narrowing (unchanged).
- test-pagebreak-variations.tex: 16 pages (+1), 0 errors, 0 body-text overlaps, 2 ghost narrowing + 2 hollow carry-over (unchanged counts, known limitations).
- test-stress-50.tex: 50 pages (+13), 0 errors. **72 body-text overlaps → 0 (ELIMINATED)**. FIGURE BESIDE TEXT 4 → 1. Quality 0.0% → 98.0%.
**Standard tests**: No regressions. All zero body-text overlaps confirmed.
