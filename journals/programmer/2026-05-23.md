### 05:00 UTC+8 — Investigation: multi-paragraph parshape extension
**Task**: Investigated Task #166 (itemize parshape leak) and the 56 body-text overlaps on 50-figure stress test.
**Finding**: All 56 overlaps are from consecutive figures WITHOUT section breaks — NOT from itemize. The root cause is that LuaLaTeX's \everypar is NOT re-entrant (fires once per paragraph, consumed by LaTeX2e). The v3.18 everypar restart trick only applies parshape to the first paragraph.
**Attempted fix (v3.20, reverted)**: Replaced everypar with a \par patch. Set \gdef\par{...} to re-set parshape for subsequent paragraphs. Result: 56 to 60 overlaps (worse). The \par patch requires a first paragraph complete guard flag to avoid regressions.
**Key tests**: everypar re-injection tested — does NOT work in LuaLaTeX. aftergroup re-injection tested — does NOT fire for next paragraph. \par patch tested — works but needs guard flags.
**Outcome**: v3.16 retained. No code change. Standard tests pass (0 errors). 56 body-text overlaps remain.
