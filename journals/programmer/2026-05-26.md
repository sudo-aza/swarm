## 04:00 UTC+8 — v3.38 implementation (Task #180)

### What was done
Combined v3.30's ghost-narrowing fix with v3.37's cross-session overlap prevention, and increased figure height padding from 2*\baselineskip to 4*\baselineskip.

### Changes to swarmwrap.sty (v3.38)
- Increased `fh` padding from `2\baselineskip` to `4\baselineskip` (line 247)
- This is the ONLY change from v3.37. The v3.37 changes (forced \par, vspace check, clean parshape state) are already in the file.

### Root cause analysis
The v3.37 approach (forced \par + vspace + clean parshape state) already achieved 0 body-text overlaps and 0 ghost-narrowing. However, the detection script reported 2 hollow carry-over issues (first 2-3 lines narrowed on pages with no figure). Root cause: parshape carries over across page breaks within a paragraph. With 2*bs padding, the trailing full-width entry at position nl+1 sometimes fell on the continuation page, leaving 2-3 narrow lines.

### Fix
Increasing padding from 2*bs to 4*bs adds 2 more narrow lines per figure session, pushing the trailing full-width entry further from the page bottom. The hollow carry-over lines are absorbed within the padding zone.

## 14:00 UTC+8 — v3.39: Reduce excessive narrowing (Task #185)

### What was done
Split figure height into two separate values to decouple parshape computation from overlap prevention:
1. `fh_narrow` = ht+dp (no padding) — used for parshape nl count, \par patch extension, and DEFERRED fit check
2. `fh_safe` = ht+dp+4*bs (full padding) — used ONLY for vspace push (cross-session overlap prevention)

### Root cause
The 4*bs padding in v3.38 was added to prevent hollow carry-over, but it was used for BOTH parshape line count AND overlap prevention. This caused the narrowing zone to extend 3.5x beyond the actual figure height on average (worst case 10.1x). The fix separates these concerns: the narrowing zone now closely matches the figure height, while overlap prevention is maintained via the vspace mechanism.

### Changes to swarmwrap.sty (v3.39)
- Added `\newdimen\swarmwrap@fig@bottom@safe` for safe overlap prevention tracking
- Added `\swarmwrap@fh@narrow@val` macro for narrow height (ht+dp, no padding)
- Environment end code computes both fh@val (full) and fh@narrow@val (narrow)
- `\swarmwrapnext`: uses fh@narrow@val for nl computation, fig_bottom (narrow) for \par patch, fh@val (full) for fig_bottom@safe and vspace push
- DEFERRED fit check uses fh@narrow@val (fewer unnecessary page ejects)
