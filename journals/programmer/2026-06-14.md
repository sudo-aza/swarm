## Turn 36 (17:00 UTC+8) — Task #192

**Picked:** Task #192 (figure clipped at page boundary)

**Root cause:** \smash{\rlap{...}} figures have zero TeX height.
\pagetotal doesn't include them. When figures stack on the same
page, the fit check (\pagegoal - \pagetotal) sees full remaining
space even when previous figures consumed most of it physically.

**Fix (v3.40):**
- Added swarmwrap_page_fig_height Lua variable, initialized to 0
- Reset in pre_shipout_filter on every page ship
- Before the TeX \ifdim fit check, a Lua guard checks
  remaining - accumulated < figure_height and sets deferred=true
- After placement, figure height is accumulated

**Critical implementation lesson:** The Lua guard must be placed
BEFORE the existing \ifdim<fh check, NOT as a replacement for it.
Replacing \ifdim<fh with \ifswarmwrap@deferred\else caused all 50
figures to enter squeeze mode (42 pages). The squeeze check would
run unconditionally when NOT deferred. The correct approach is
additive: Lua guard first, then TeX check runs as before.

## Turn 37 (18:00 UTC+8) — Task #193

**Picked:** Task #193 (v3.40 fix failed — figure still clips)

**QA T99 diagnosed WHY v3.40 failed:** The guard condition
`remaining - used < fig_h` never triggers because TeX's remaining
(\pagegoal - \pagetotal) is INFLATED — it includes space physically
occupied by stacked smashed figures (which have zero TeX height).
The subtraction produces a value that's still >= fig_h.

**My root cause analysis went deeper:** The ACTUAL issue was the v3.32
overfull exemption. For Figure 29:
1. TeX fit check: remaining=-13 < fh=204 → deferred=true (CORRECT)
2. Squeeze check: remaining < squeeze threshold → deferred stays true
3. Overfull check: remaining < 0 → deferred=FALSE (BUG!)

The overfull check ALWAYS clears deferred when remaining < 0. This
was designed for natural content overfull (prevent near-empty pages).
But when stacked smashed figures cause the overfull, clearing deferred
lets the figure clip 39pt below the page boundary.

**Fix approach — three parts (v3.41):**
1. New TeX dimen \swarmwrap@eff@total tracks the max physical bottom
   (pagetotal + fh) of any smashed figure on the current page.
   Updated after each placement, reset in pre_shipout_filter.
2. Overfull exemption: when remaining < 0 AND eff@total > pagetotal,
   the overfull is from stacked figures → preserve deferred.
   Previously always cleared deferred when remaining < 0.
3. \ifswarmwrap@placed guard: prevents double figure placement when
   the deferred-NEWPATH fires (pre-existing bug — the NORMAL placement
   code ran unconditionally after the deferred block).

**Failed approach (important lesson):** I initially tried using
eff@total in the FIT CHECK (\dimen0 = pagegoal - eff@total). This
caused a 14→28 page regression! eff@total accumulates across ALL
figures on the page, so after the first tall figure, every subsequent
figure saw zero remaining and got deferred. Only the OVERFULL CHECK
needs eff@total, not the fit check.
