## 14:00 UTC+8 — Task #56: Test figflow (plain TeX)

**Package**: figflow (Ian Hutchinson, 1995) — plain TeX \hangindent approach
**Test file**: `src/test-wrapfig/test-figflow.tex`

### API Notes
- `\figflow{width}{height}{content}` — negative width = right, positive = left
- Height = total figure height including caption
- Auto-ends when `\prevgraf > \figlines` — do NOT call `\endflow` manually
- Source says "Does not work for Latex" — `\line` conflicts with LaTeX picture mode
- Workaround: `\let\LaTeXline\line \def\line{\hbox to\hsize}`
- Must load with `\input{figflow}` (plain TeX macro, not LaTeX package)
- Uses `\hangindent`/`\hangafter` (not parshape), redefines `\par`

### Test Results (6 tests)
| Test | Description | Result |
|------|-------------|--------|
| 1 | Right figure | PASS |
| 2 | Left figure | PASS |
| 3 | Tall figure, page break | PASS (deferred, not overflowed) |
| 4 | Figure before itemize | PARTIAL PASS (missing \item error) |
| 5 | Figure inside itemize | FAIL (figure collision) |
| 6 | multicol | FAIL (figure collision, group leak) |

### Key Finding
figflow is the ONLY package tested so far that handles page breaks gracefully — it detects insufficient room via `\pageremains` and defers the figure to the next page instead of overflowing. However, the auto-end mechanism is fragile: if there aren't enough paragraphs before the next `\figflow` call, `\flevel` leaks and causes "Figure collision" for all subsequent figures. The `\begingroup` can also leak, producing group warnings.

## 15:00 UTC+8 — Task #57: Test shapepar (v2.2)

**Package**: shapepar (v2.2, Donald Arseneau 2013) — parshape-based decorative paragraph shapes
**Test file**: `src/test-wrapfig/test-shapepar.tex`

### API Notes
- `\shapepar[<scale>]{<shape_spec>} text \par` — shapes a single paragraph via parshape
- `\Shapepar` (capital S) = boxed version, no centering, for float contexts
- `\cutout{r|l}(h_off, v_off)` — precedes `\shapepar`, wraps subsequent paragraphs around shape
- `\cutoutsep` = gap between shape and cutout text (default 12pt)
- Predefined shapes: `\squareshape`, `\circleshape`, `\diamondshape`, `\heartshape`, `\starshape`, `\nutshape`
- Convenience: `\squarepar{text}`, `\diamondpar{text}`, etc.
- CRITICAL: NO vertical material inside shaped paragraphs (`\vspace`, `\\`, `\centering` all forbidden)
- Images cannot be placed inside shaped paragraphs — shapepar is for text-only decorative shapes

### Key Discovery
`\circlepar` causes arithmetic overflow in some contexts due to small coordinate values (0-1 range). `\diamondpar` is numerically stable. `\cutout` + `\Shapepar` + `\rectangleshape` fails because `\cm` units inside shape spec conflict with shapepar's internal parsing — use predefined shapes instead.

## 16:00 UTC+8 — Task #58: Test paracol (v1.37)

**Package**: paracol (v1.37, Hiroshi Nakashima / Markus Kurtz) — parallel independent columns
**Test file**: `src/test-wrapfig/test-paracol.tex`

### API Notes
- `\begin{paracol}{ncols} ... \end{paracol}` — start parallel column environment
- `\switchcolumn` — switch to next column (wraps around)
- `\switchcolumn*[<span>]` — sync all columns + optional spanning text
- `\columnratio{0.4}` — left column 40%, right 60%
- `\globalcounter{figure}` — share figure counter across columns
- Column environments: `\begin{leftcolumn}`, `\begin{rightcolumn}`, `\begin{nthcolumn}{n}`
- `\flushpage` — flush all columns, start fresh page
- NOT compatible with multicol as a replacement (different approaches)
- Compatible with itemize/enumerate inside columns

### Key Difference from Other Packages
paracol uses independent parallel columns, NOT parshape. This is a fundamentally different approach — the figure occupies one full column while text flows in another column. There is no gradual text width transition as with wrapfig/cutwin/insbox. The visual effect is a rigid column boundary, not organic text wrapping.
