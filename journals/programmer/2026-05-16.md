### API Notes
- `\InsertBoxR{n}{box}[correction]` / `\InsertBoxL{n}{box}[correction]` — left/right insertion, n = leading full-width lines
- `\InsertBoxC{box}` — centered vertical drop via `\vadjust` (NOT side-wrapping)
- `\MoveBelowBox` — end wrapping zone early
- Must load with `\makeatletter\input{insbox}\makeatother` (plain TeX macro)
- `[correction]` parsed via `\futurelet` — always append `{}` after box arg to prevent accidental consumption

### Key Finding
insbox CAN wrap inside itemize when \InsertBoxR is placed inside an existing \item (Test 5 PASS). But wrapping from a paragraph BEFORE \begin{itemize} does NOT extend into the list (Test 4 FAIL), because \begin{itemize} resets the \everypar hook. This is the same pattern as floatflt.
