## Turn 04:00 UTC+8 — Task #178: Fix cross-session body-text overlaps (v3.34)

### Task Picked
Task #178 — Fix 91 cross-session body-text overlaps on 50-figure stress test (v3.30 QA review, Task #177 FAIL)

### Analysis
Read programmer-rules.md, BLACKBOARD.md, programmer-verification-guide.md, wrapping-specs.md, and QA journal (2026-05-24).

**Root cause identified**: v3.30's trailing full-width parshape entry (`0pt \linewidth` at position nl+1) resets text to full page width after nl narrow lines. Since figures are placed via `\smash{\rlap}` (invisible to TeX layout), TeX allows full-width text to occupy the same horizontal space as the visually-present figure. This caused 91 body-text overlaps on the 50-figure stress test.

Additionally, the `\par patch` had trailing full-width entries that allowed subsequent paragraphs to go full-width after the figure zone ended, contributing to cross-session overlaps.

### Implementation (v3.34)
Three changes to `src/themes/swarmwrap.sty`:

1. **Removed ALL trailing full-width parshape entries** from: `\swarmwrapnext` (main path), `\par patch`, list hook (`\swarmwrap@list@everypar@hook`), and item hook (`\swarmwrap@apply@item@parshape`). All lines within a paragraph now stay narrow — TeX repeats the last narrow entry for subsequent lines.

2. **\par patch NEVER deactivates on the same page as the figure**. When `\pagetotal >= fig_bottom`, instead of deactivating, the patch now computes remaining lines to page goal and extends narrowing for that many lines. Only deactivates on a new page (`fig_page != current_page`).

3. **Increased figure height padding** from `2*\baselineskip` to `4*\baselineskip` for extra vertical clearance.

Updated Lua callback (`swarmwrap-callback.lua`) version string to v3.34.

### Cross-session width guard (ATTEMPTED AND ABANDONED)
Initially implemented a cross-session width guard in `\swarmwrapnext` that forced a page break when a new session's tw was narrower than the previous session's tw (indicating a wider figure). This was too aggressive: 58 pages, 493 overlaps, 17 FIGURE BESIDE TEXT. Removed in favor of the "narrow to page end" approach.

**Root cause analysis**:
The fundamental architecture uses `\smash{\rlap{...}}` to place figures
invisibly to TeX's layout engine. This means TeX cannot account for the
figure's vertical extent when placing subsequent content. The parshape mechanism
controls line width but NOT page-level layout.

When the trailing parshape entry is full-width (v3.30), TeX repeats it for all
subsequent lines, fixing ghost-narrowing on continuation pages. But the
full-width text from session N's trailing entry overlaps session N+1's figure
on the same page (cross-session overlap).

When the trailing entry is narrow (v3.34), there are no overlaps but TeX repeats
the narrow entry for ALL subsequent lines including continuation pages, causing
ghost-narrowing.

The two goals (0 ghost-narrowing AND 0 overlaps) require DIFFERENT parshape
behaviors on the same page — narrow beside the figure, full-width elsewhere.
TeX's parshape is per-paragraph and cannot change mid-paragraph. Lua callbacks
cannot modify glyph positions after line breaking (post_linebreak_filter,
pre_shipout_filter, hpack_filter all fail for different reasons).

## Turn 19:00 UTC+8 — Task #180 continued: 3 more approaches, all blocked

**Objective**: Same as 15:00 turn — fix both ghost-narrowing AND overlaps.

**Approaches tried (all failed)**:

1. **Global figure zone tracking + cross-session nl extension**: Added `global_fig_bottom` and `global_fig_page` registers to track the furthest figure bottom across ALL sessions on a page. In `\swarmwrapnext`, saved previous `global_fig_bottom` before updating, then computed extended `nl` if previous zone extends further than current session's zone. Verified triggering: debug logs confirmed nl extended from 9→27 for sessions with small figures after tall ones. **LIMITATION**: Only prevents FORWARD overlaps (current session's text overlapping FUTURE figures). BACKWARD overlaps (previous session's already-typeset full-width text overlapping CURRENT figure) remain because the text was already typeset before the current session started. Result: unchanged at 91 overlaps.

2. **Page eject guard before computation**: Added check at start of `\swarmwrapnext`: if `pagetotal < global_fig_bottom` on same page, eject to new page before placing new figure. **FAILED**: `pagetotal` does not track `\smash{\rlap}` content. After deferred path's `\newpage`, `pagetotal` resets to 0 but `global_fig_bottom` still holds previous page's value. Guard fires on every subsequent session. Result: 66 pages, 644 overlaps, 20 no-wrap figures — much worse than baseline.

3. **`\vadjust` phantom rule**: Insert `\vadjust{\vrule width0pt height\fh depth0pt}` after `\smash{\rlap}` figure placement to make TeX's page builder aware of figure height, which would make `\pagetotal` track figure zones for the page eject guard. Minimal test worked, but stress test caused stack overflow (`input stack size=10000`). The `\vadjust` interaction with `\smash{\rlap}` and conditional branches in `\swarmwrapnext` caused infinite macro expansion.

**Additional insight from debugging**:
- On page 1 (before any DEFERRED), `pagetotal` correctly tracks text height (335-348pt per session). The cross-session extension condition (`dimen2 = prev_global_bottom - pagetotal`) is negative because text extends past previous figures. No extension needed.
- After first DEFERRED, `pagetotal` shows 0.0pt for ALL subsequent sessions (on the same page). `\c@page` shows the same page number (5) for all of them. This means TeX is shipping pages between sessions but the page counter isn't updating at `\swarmwrapnext` execution time.
- The `prev_global_bottom` value (360.45pt) is carried across multiple sessions on the same page. Sessions with smaller figures (nl=9) correctly get extended to nl=27 to cover the gap. But this extension is forward-only.

**CONCLUSION**: Confirmed 15:00 analysis — the `\smash{\rlap}` architecture fundamentally cannot achieve zero ghost-narrowing AND zero cross-session body-text overlaps simultaneously. Total approaches tried across 15:00 and 19:00 turns: 9, all blocked by the same architectural limitation.
