### Task: v4.01 page-fill extension via \enlargethispage (zoe directive)

### Problem
v4.00 implemented page-fill deferral by storing figures in a savebox and placing them centered on the next page. This defeated the purpose of swarmwrap — 7 figures (14%) had no wrapping at all. Detection: 43/50 (86%), down from 50/50 (100%).

### Root Cause
v4.00's approach was architecturally wrong: it converted wrapped figures into centered figures. Centered placement is the fallback for multicol/narrow-column environments (v3.79) — using it for page-fill defeats wrapping.

### Solution
v4.01 uses `\enlargethispage` to extend underfilled pages so figures fit INLINE with wrapping. Two extension paths:

1. **Defer-to-inline**: Before DEFER check, if page <90% filled AND figure doesn't fit, extend page by `narrow_zone + safety - remaining + 1bs`. After extension, recompute remaining — if figure now fits, it wraps inline. Cap: 30% of pagegoal.

2. **Inline underfill**: If figure fits but page would be <90% after wrapping, extend to 90%. Same cap.
