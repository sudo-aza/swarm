### Task: #278 — Near-empty page prevention in DEFER path (v3.93)

**Context**: Zoe reported page 52 has half the page empty. Previous turn (Task #275) reduced DEFER margin from 6bs to 3bs, which helped but left 33 near-empty pages on 1000-fig. Root cause analysis this turn: 30 of 33 near-empty pages are caused by DEFER ejecting an overfull page. When remaining < 0 (page overfull by ~1 line), TeX's `\newpage` ships the page content but the overflow line (1 narrow carry-over line) goes to a NEW page, creating a near-empty page.

**Investigation**:
- Traced figures 48-58 in wlog: Fig 52 DEFER with remaining=-13.6pt. The page was overfull by 1 line.
- Checked page 52 in PDF: 1 narrow line (w=274pt) at [118,125]-[392,141], then empty until page number at y=738. Figure 53 on page 53 (skipped page 52).
- Analyzed all 33 near-empty pages: 32 have exactly 2 text blocks (1 text line + page number), 0 figures. 1 (page 142/137) has 5 blocks, 1 figure (small itemize).
- Counted DEFER entries with remaining < 0: 30 entries all with remaining=-13.6pt. These correspond to the 32 near-empty pages.
- Investigated why `\pagetotal` check failed: at DEFER time, TeX hasn't shipped the page yet (emergencystretch absorbs the -13.6pt overflow), so `\pagetotal` is still ~713pt (full page). Must check `\dimen0` (remaining) instead.

**Approach**: In the DEFER path (before `\newpage`), check if remaining < 0. If so, use `\enlargethispage{-\dimen0}` to enlarge the page goal by the overflow amount. This absorbs the carry-over line into the current page, so `\newpage` ships cleanly with no overflow.

**Changes**:
1. `src/themes/swarmwrap.sty` v3.93:
   - Added `\ifdim\dimen0<0pt\relax` check before `\newpage` in DEFER path (line ~1026)
   - When true: `\enlargethispage{-\dimen0}` + wlog entry
   - Fixed bare `#274` in `\wlog` inside `\newcommand{\swarmwrapnext}` (line 909) causing "Illegal parameter number" error
   - Updated stale comment line 377: "6\baselineskip" → "3\baselineskip"
   - Updated `\ProvidesPackage` version string to v3.93
