# Programmer Journal — 2026-05-27

## Turn: v3.51 Ghost-narrowing fix (Task #199/#204 follow-up)

### Task
Fix remaining ghost narrowing at 1000-fig scale. v3.50 achieved 0 ghost narrowing on 50-fig test but 3 ghost narrowing pages (67, 332, 927) persisted at 1000-fig scale.

### Root Cause Analysis
1. **Penalty 10000 is not truly prohibitive**: In TeX, penalty > 10000 is "prohibitive" — TeX will never break at that point unless the page is completely overfull with no alternative. Penalty 10000 is only "very discouraged" and CAN be overridden by TeX when alternatives are worse (e.g., long paragraph forcing overfull vbox).
2. **Page-space limit has no safety margin**: The \par patch computes `page_remaining_lines = (pagegoal - pagetotal) / baselineskip` but doesn't account for \parskip, \lineskip, or other vertical glue. This can cause the narrow zone to extend to the exact page boundary, leaving no room for the trailing full-width entry.
3. **Narrow-line detection tolerance too tight**: 3.0pt tolerance missed lines with \emergencystretch that had width > tw+3.0pt.
4. **Phase 2 glue_set widening is broken**: Changes hbox.width and glue_set but does NOT reposition text glyphs — same class of bug as v3.45 (QA verified).

### Changes Made
1. **swarmwrap.sty v3.51**:
   - Default penalty: 10000 → 10001 (truly prohibitive)
   - Added `\advance\dimen2 by -\baselineskip` safety margin in 3 places: \par patch, list hook, item hook
   - Updated version header to v3.51
   - Updated Known Limitations and Changelog documentation

2. **swarmwrap-callback.lua v3.51**:
   - Narrow-line detection tolerance: 3.0pt → 10.0pt
   - Added `w < lw_val * 0.95` check to avoid flagging full-width lines as narrow
   - Removed Phase 2 glue_set widening code entirely (lines 80-134 of v3.50)
   - Simplified to penalty-only callback
