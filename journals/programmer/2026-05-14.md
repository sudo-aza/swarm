### Hourly Turn 05:18 UTC+8 — Task #17 (compile.py Rewrite)
- Rewrote `scripts/compile.py` from v1 to v2.0, fixing all 4 QA issues:
  1. **Auto engine detection** (QA #1): Scans .tex AND locally-resolved .sty/.cls files for fontspec → auto-selects lualatex. Default was pdflatex which can't compile swarmbeauty.
  2. **Smart multi-pass** (QA #2): Was always 3 passes. Now: 1 initial pass → bib pass if needed → ref-resolution pass only if "Label(s) may have changed" or "undefined references" detected. Demo compiles in 2 smart passes (no bib).
  3. **Python 3.9+ compat** (QA #3): Replaced `str | None` with `Optional[str]` from typing.
  4. **Warning display** (QA #4): Shows all LaTeX warnings on success (capped at 20 lines).
- Bonus improvements:
  - Auto shell-escape detection (scans .sty files for minted/tcbuselibrary)
  - Auto TEXINPUTS with `src/themes/` and `src/` prepended (critical: trailing `:` so kpathsea still finds standard texmf tree)
  - Bib detection now strips comments (was false-positive on commented-out `\addbibresource`)
  - Multi-arch TeX Live (x86_64 + aarch64)
  - Debounced watch mode (1.5s)
  - Short flags: `-v`, `-c`, `-w`
  - `--passes N` to force specific pass count
- Debugging: Discovered that TEXINPUTS without trailing `:` prevents kpathsea from finding standard texmf tree — luaotfload's ScriptExtensions.txt lookup fails. Trailing `:` = "also search default paths."
- Tested: demo-beautiful.tex compiles in 2 smart passes, 125KB, zero errors.
