### Hourly Turn 08:00 UTC+8 — Task #24 (TOC Styles Not Applying — 3 Root Causes)
- Pulled latest: QA revoked 10/10 rating on Task #23 after installing TeX Live and doing ACTUAL visual QA (not just code review). TOC entry styles were silently not being applied. Rating revised to 5/10. Created Task #24.
- Investigated with PyMuPDF: extracted actual font/color data from compiled PDF to compare against spec.
- **Root Cause 1 — titlesec + KOMA conflict**: `titlesec` redefines `\l@section` which overrides tocbasic's `\DeclareTOCStyleEntry` entryformat/entrynumberformat/pagenumberformat. KOMA itself warns about this on every compile (5 warnings). **Fix**: Removed `titlesec` entirely. Replaced with:
  - `\setkomafont{section/subsubsection}` for fonts
  - `\RedeclareSectionCommand` for beforeskip/afterskip spacing
  - Custom `\sectionlinesformat` redefinition for decorative rules after section/subsection headings (uses `\ifstr{#1}{section}` to conditionally add rules)
- **Root Cause 2 — hyperref linkcolor**: Even without titlesec, ALL TOC text was `#3498DB` (linkcolor) instead of the entryformat colors. hyperref wraps TOC entries in `\hyperlink` which applies `linkcolor` AFTER entryformat. **Fix**: Changed `linkcolor = sbSecondary` to `linkcolor = .` (inherit current color). External URLs/citations still colored via urlcolor/citecolor.
- **Root Cause 3 — AutoFakeBold**: `\setsansfont{Latin Modern Sans}[AutoFakeBold=2]` makes fontspec use Regular font with synthetic emboldening. PDF font name remains "LMSans10-Regular" even when `\bfseries` is requested. **Fix**: Removed `AutoFakeBold=2` and `AutoFakeSlant=0.2` from all three font settings. Latin Modern has real bold/italic variants.
- **Also fixed**: `\setkomafont{sectionnumber}` etc. are NOT valid KOMA font elements — caused errors. Removed them.
- **Methodology**: Used PyMuPDF (like QA) to verify actual PDF output. Created minimal test document to isolate issues.
- Results verified via PyMuPDF extraction: section entries LMSans10-Bold #2C3E50, subsection LMSans10-Regular #34495E, numbers #3498DB, page numbers #3498DB, leaders #BDC3C7, section rules #3398DB, subsection rules #BDC3C7, body headings LMSans10-Bold #2C3E50.

### Hourly Turn 09:00 UTC+8 — Task #30 (TOC Layout — Uniform Spacing + Tighter numwidth)
- Pulled latest: QA revised v0.4.0 rating from 10/10 to 7/10 after Robby_Blue pointed out TOC spacing issues in the actual screenshot. QA found vspace between TOC entries ranging from 24pt to 76pt, and numwidth=2.5em wasting space for single-digit section numbers.
- **Root cause analysis**: The global `parskip=half-` setting (from `\KOMAoptions{parskip=half-}`) is inherited by TOC entries, which are internally rendered as paragraphs. This variable paragraph skip combined with default KOMA TOC spacing creates the wildly inconsistent gaps.
- **Fix 1 — Uniform vspace**: Added `\BeforeStartingTOC[toc]{\setlength{\parskip}{0pt plus 0pt}}` to kill parskip inside the TOC. Set explicit `beforeskip` values per entry level: section 0.8em plus 0.2em, subsection 0.3em plus 0.1em, subsubsection 0.2em plus 0.1em. This gives LaTeX controlled stretch but eliminates the massive variable parskip.
- **Fix 2 — Tighter numwidth**: Reduced section numwidth from 2.5em→1.5em (for "1" through "8" entries), subsection from 3em→2.5em (for "1.1" through "8.2"), subsubsection from 3.5em→3.0em (for "1.1.1" entries).
- **Verification via PyMuPDF Y-coordinate analysis**:
  - Section→subsection gaps: 15.7-15.8pt consistently (range: 0.1pt)
  - Section→section gaps: 21.4pt consistently (range: 0pt)
  - Subsection→subsection gaps: 15.7pt consistently
  - Title→first entry: 43.3pt (intentional, good visual separation)
  - Previous: 24pt to 76pt range → now: 15.7pt to 21.4pt range (two distinct, consistent values)
- **Verification via PyMuPDF X-coordinate analysis**:
  - Section number-to-title gap: 16.3pt (was ~21pt)
  - Subsection number-to-title gap: 27.3pt (reasonable for "1.1" format)
  - Indentation correct: sections at x=68.7, subsections at x=85.0 (1.5em indent)
