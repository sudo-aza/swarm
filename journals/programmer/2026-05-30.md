**Root cause analysis:**
The v3.17 list patch checked `\ifnum\swarmwrap@remaining@nl>0` before reducing `\linewidth` for list environments. If any text appeared between `\swarmwrapnext` and `\begin{itemize}`, the first paragraph consumed the narrow-line budget, setting `remaining@nl` to 0. This explains why v3.18 only improved from 66 to 41 overlaps — the patch worked when `\swarmwrapnext` was immediately followed by a list, but failed when text came first.

### Task: Ghost-narrowing page 67 investigation — DEFER margin increase (Task #214)

**What changed:**

- swarmwrap-callback.lua v3.67: Version string update only

**Investigation approach:**
Systematically tested 3 margin configurations to isolate whether the ghost narrowing on page 67 is caused by insufficient safety margins:
1. DEFER 5bs→8bs, page-space 5bs: 1096/1100 (99.6%) — 2 ghost
2. DEFER 8bs, page-space 8bs: 1096/1100 (99.6%) — 2 ghost (identical)
3. DEFER 8bs, page-space 12bs: 1096/1100 (99.6%) — 2 ghost (identical)

All 3 configurations produced byte-identical detection results. This proves the ghost narrowing is NOT caused by insufficient margins in either the DEFER check or the \par patch page-space check.

**Root cause analysis:**
The ghost narrowing on page 67 comes from TeX's \parshape persisting within a SINGLE paragraph across page breaks. When a paragraph starts with narrow parshape entries and the paragraph is long enough to span a page break, TeX carries the remaining narrow entries to the next page. This is an inherent TeX limitation:
- The DEFER check prevents figure placement when the narrow zone doesn't fit, but the narrow zone DOES fit (passes DEFER check)
- The page-space check limits \par patch extensions, but the ghost is from the INITIAL parshape, not extensions
- No margin-based approach can prevent a single paragraph from spanning a page break

The 8bs DEFER margin correctly deferred one additional figure (nl=22, remaining=385 < 407pt), confirming the mechanism works. But the ghost-causing figure has a smaller nl whose narrow zone easily passes all checks.

**Conclusion:**
Eliminating the remaining 2 ghost pages (pages 67 and 903) requires a fundamentally different approach: **Approach A — Split-and-Re-Linebreak via `tex.linebreak()` in `post_linebreak_filter`**. This technique is proven by the `lua-widow-control` package (TUGboat 43-1, 2024). It would detect when a paragraph's narrow zone would carry over across a page break and force a page break at the narrow→full transition point. See `notes/2026-05-28-paragraph-splitting-research.md` for implementation details.

Page 903 is likely a multicols false positive (per QA's Task #221 analysis showing multicols paragraph structure in span analysis).

**Implementation approach:**
Registered `linebreak_filter` callback that replaces TeX's paragraph builder. The function:
1. Checks if wrapping is active (nl > 0); if not, delegates to `tex.linebreak(head, display)`
2. Calls `tex.linebreak(head, display)` to break lines normally
3. Walks the broken paragraph to find the narrow->full transition
4. Estimates remaining page space (pagegoal - pagetotal)
5. If narrow zone > 60% of page space AND >= 3 narrow lines, inserts forced page break (penalty -10000 + vfill) at the transition
6. Otherwise inserts default transition penalty (-2000)
Safety: pcall wrapper, recursion guard, delegation for display math and non-wrapped paragraphs.

**Implementation approach (Approach A per research note):**
Task #224 asked to implement Approach A via `post_linebreak_filter`. The research note describes two options:
- Option 1: "Force page break before this paragraph (simplest)" — conditional on remaining page space
- Option 2: "Split node list and re-linebreak (complex but elegant)"

Option 2 is **impossible** in `post_linebreak_filter` because the callback receives the already-broken paragraph (vlist of hlists), and the original unbroken node list is no longer available. `tex.linebreak()` requires the unbroken list.

Option 1 was implemented but the conditional check (`para_height > remaining`) never triggered because at `post_linebreak_filter` time, individual paragraphs always fit within the remaining page space. The ghost narrowing occurs when the CUMULATIVE page content overflows, not individual paragraphs.

As a fallback, the penalty was changed to unconditional -5000 (stronger than previous -2000, but below forced-break threshold of -10000). However, due to LuaTeX paragraph caching with `\lipsum`, the result is byte-identical to -2000 — TeX breaks at the same positions regardless.

**Key finding: Penalty-based approaches in `post_linebreak_filter` have the same limitation as all previous penalty approaches.** The penalty is inserted between hlists within the paragraph vlist. TeX's page builder may or may not respect these penalties depending on the overall page layout. For the 2 ghost-narrowing edge cases, TeX's page breaker overrides the penalty because the page is overfull.

**Key innovation:**
Previous penalty approaches used discretionary penalties (-2000 v3.54, -5000 v3.69) that TeX can override when the page is overfull — exactly when ghost narrowing occurs. v3.70 uses -10000, which is a FORCED break in TeX (TeX MUST break at this point, it cannot override). **Technical challenge solved:**
`tex.dimen["pagegoal"]` and `tex.dimen["pagetotal"]` are TeX internals NOT directly accessible from Lua callbacks (confirmed by test: all 4 access methods fail with "incorrect dimen name"). Solved by computing remaining at \swarmwrapnext time and storing in a user-defined \newdimen register that IS Lua-readable.

Key finding: No threshold tuning can reduce forced breaks enough. With `\lipsum` (LuaTeX paragraph caching), the penalty is computed once per unique shape and applied to ALL occurrences.
