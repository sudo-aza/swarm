### Task: Fix itemize list patch page-boundary check (Task #166)

**What changed:**
- swarmwrap.sty v3.27: Fixed list patch (`\renewcommand{\list}`) to use `tw@lua > 0` + `fig@page == \c@page` check instead of `remaining@nl > 0`
- Added `\newcount\swarmwrap@fig@page` counter (line 330)
- Set `\global\swarmwrap@fig@page=\c@page` in `\swarmwrapnext` (line 478)
- Updated comments explaining v3.27 fix rationale

**Root cause analysis:**
The v3.17 list patch checked `\ifnum\swarmwrap@remaining@nl>0` before reducing `\linewidth` for list environments. However, `post_linebreak_filter` decrements `remaining@nl` after each paragraph's lines are broken. If any text appeared between `\swarmwrapnext` and `\begin{itemize}`, the first paragraph consumed the narrow-line budget, setting `remaining@nl` to 0. When `\begin{itemize}` started, the list patch saw `remaining@nl = 0` and didn't reduce `\linewidth`, so list items ran at full width through the figure. This explains why v3.18 only improved from 66 to 41 overlaps — the patch worked when `\swarmwrapnext` was immediately followed by a list, but failed when text came first.
