## Turn 23:00 UTC+8 — Investigation: body-text overlaps root cause

### Task
Task #164: Investigate and fix body-text overlaps on 50-figure stress test.

### Setup
- Installed TeX Live (LuaHBTeX 1.24.0) — setup.sh succeeded, network recovered
- Merged remote changes (git pull resolved conflicts in upload files)
- Baseline: v3.14, 37 pages, 60 body-text overlaps, 4 FBT, 0 ghost

### Investigation Findings

**Root cause of 60 overlaps: NOT everypar-related.**

1. `\lipsum[1]\lipsum[2]` on the same line produces ONE paragraph in TeX, not two. The `\lipsum` command does NOT end with `\par` when followed by more tokens on the same line. Only `\lipsum[1]\par\lipsum[2]` (explicit `\par`) produces two paragraphs.

2. TeX's `\everypar` is a one-shot mechanism — tokens are consumed after firing for one paragraph. The hook must be explicitly re-armed (`\global\everypar{...}`) at the end of each invocation.

3. Even with self-rearming everypar (v3.15 attempt), overlaps were NOT reduced. The everypar only fired 5 times for 49 figures because most figures have only one paragraph (`\lipsum[1]\lipsum[2]` = one paragraph).

4. The 60 overlaps come from FULL-WIDTH text AFTER the narrow parshape lines, extending through the NEXT figure's area. The parshape provides `nl = ceil(fh / baselineskip)` narrow lines. When the paragraph is long enough to use all `nl` lines, the remaining text runs at full width and may overlap the next figure placed below.

5. The figure is placed via `\smash{\rlap{...}}` at the start of the narrow zone. If `nl * baselineskip > fh`, the narrow zone covers the full figure height. But the full-width text AFTER the narrow zone may still overlap figures on subsequent pages or figures placed by later `\swarmwrapnext` calls.

### Attempted Fix (v3.15, reverted)
- Removed remaining counter decrement in post_linebreak_filter
- Made everypar global and self-re-arming
- Did NOT reduce overlaps — everypar mechanism doesn't address the root cause
- Reverted to v3.14 (commit)

### Key Insight
The overlaps are NOT caused by consecutive figures clearing each other's everypar. They're caused by long paragraphs that exhaust their parshape narrow lines, with the remaining full-width text flowing through subsequent figure areas. The fix must either:
1. Ensure narrow lines cover the full figure height even in long paragraphs
2. Prevent full-width text from overlapping figure zones (Lua shipout_filter)
3. Use a fundamentally different approach than parshape for multi-line text

### Next Steps
- Investigate using Lua `pre_linebreak_filter` to dynamically extend parshape
- Or use Lua `shipout_filter` to detect and prevent text-figure overlap
- Consider if Robby's "reduce nl by 1" approach helps (it might reduce ghost narrowing but not overlaps)
