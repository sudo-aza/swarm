### Task: Fix multicol layout issue — v3.79 (Task #245)

**Problem**: Page 1 of test-stress-50.pdf has a `multicols{2}` environment
containing a swarmwrap figure (2cmx2cm, Fig 0). The result was "extremely
dense hyphenation, fragmented partial words everywhere" (zoe's feedback).
The detection script missed this because it only checks for bbox overlaps,
not typographic quality.

**Root cause**: Inside multicol, `\linewidth` is already narrow (~175pt for
2-col A4). The parshape wrapping computation `tw = linewidth - fw - 14pt`
produced ~104pt text width — too narrow for readable text with 10pt font.

**Approach**: Detect narrow-column environments by comparing `\linewidth` to
`\textwidth`. If `linewidth < 0.7\textwidth`, fall back to centered figure
placement using the existing `\swarmwrap@place@centered` helper. This
avoids parshape wrapping entirely in narrow columns.

**Why 0.7 threshold**: 
- 2-col multicol: linewidth=175pt, textwidth=360pt → 175 < 252 → TRUE (trigger)
- Single-column A4: linewidth=345pt, textwidth=360pt → 345 > 252 → FALSE (no trigger)

**Implementation**: 15-line addition to `\swarmwrapnext`, wrapped in
`\ifdim\linewidth<0.7\textwidth ... \else ... \fi`. No other files changed.

**Results**:
| Metric | v3.78 (before) | v3.79 (after) | Change |
|--------|----------------|---------------|--------|
| test-stress-50: pages | 48 | 49 | +1 (centered multicol fig) |
| test-stress-50: quality | 50/50 (100%) | 49/50 (98%) | -1 fig (multicol, intentional) |
| test-stress-1000: pages | 1069 | 1069 | 0 |
| test-stress-1000: quality | 1100/1100 (100%) | 1100/1100 (100%) | 0 |
| test-customwrap: pages | 8 | 8 | 0 |

**Key finding**: The 1 "no-wrap" figure in 50-fig is the multicol figure
