**Task**: v3.34 — Fix ghost narrowing via buildpage_filter hbox widening.

**Approach**: New `buildpage_filter` callback. When `post_linebreak_filter`
detects `total_lines < nl` (paragraph crossed page break in narrow zone),
it sets Lua globals: `swarmwrap_ghost_active=true`,
`swarmwrap_ghost_page=tex.count["c@page"]`, `swarmwrap_ghost_tw=tw_sp`.
The `buildpage_filter` then inspects every hlist node being added to pages
past the figure page. If an hbox width matches `swarmwrap_ghost_tw` within
3pt tolerance, it widens it to `\linewidth` and appends a `\hfil` glue node.
Flag cleared on next `\swarmwrapnext` call.

## Turn 2 (03:00 UTC+8, cron-triggered)

**Task**: Verify v3.47 ghost narrowing fix — QA has not checked yet.

**Result**: v3.47 does NOT fix ghost narrowing. Confirmed via:
  - detect-layout-issues.py: 3 ghost pages (3, 7, 13), 3 hollow carry-overs
  - PyMuPDF analysis: page 3 first lines at x1=377.5pt (should be 476.5pt)

**Root cause discovered**: v3.47's ghost detection code in
post_linebreak_filter was dead code. It checked remaining@nl > 0,
but buildpage_filter (v3.34) zeros remaining@nl when the page ships.
Since buildpage fires before post_linebreak on the continuation page,
the check always saw remaining@nl=0.

**Attempted v3.48 fix (REVERTED)**: Tried multiple approaches:
  1. Move flag setting to buildpage_filter → flag set 10x but consumed
     by next PLB call for a DIFFERENT paragraph (flag has no page
     affinity). Widened 179 hboxes including correct narrow lines.
  2. Save last tw and detect in PLB when tw=0 → PLB receives ALL
     lines of a broken paragraph in one call, including continuation
     lines. tw@lua is still positive for those lines because BPF
     hasn't fired yet. Dead end.
  3. BPF-based widening on continuation pages → tex.lineWidth is nil
     in BPF context. Used saved_lw from PLB via tex.dimen["linewidth"].
     Fixed 2/3 ghost pages but caused 13→14 page regression and false
     positives on pages 6 and 11.

**Key technical findings**:
  - texio.write_nl does NOT work inside node-processing callbacks
    (post_linebreak_filter, buildpage_filter). Output goes nowhere.
  - buildpage_filter fires 318 times for a 13-page document.
  - tex.lineWidth returns nil in buildpage_filter; tex.dimen["linewidth"]
    works in post_linebreak_filter but not buildpage_filter.
  - LuaTeX's post_linebreak_filter receives ALL lines of a broken
    paragraph in one call (not split by page). Cannot distinguish which
    lines will end up on the continuation page.
  - The fundamental problem: TeX's paragraph breaker applies parshape
    to ALL lines before page breaking. No Lua callback can intercept
    the continuation portion separately.

**No code changes committed** (reverted to v3.47). All 3 standard test
suites compile clean with v3.47 (0 errors).
