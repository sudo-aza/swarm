## 04:00 UTC+8 — Task #117: swarmwrap.sty v3.0 zero-arg rewrite

**Files**: `src/themes/swarmwrap.sty`, `src/test-wrapfig/test-customwrap.tex`
**Origin**: zoe scope reduction — simplest possible right-side float wrapper.

### Problem

The v2.5 API required a mandatory width argument: `\begin{swarmwrap}{3cm}...`. This forced a fixed-width minipage inside the environment, which was redundant since the content (e.g., `\includegraphics[width=3cm]`) already knows its own width. The task asked for a zero-argument API where both width and height are auto-detected.

### Approach

Removed the `[1]` argument from `\newenvironment{swarmwrap}`, removed the fixed-width minipage from begin/end code, and added width auto-detection (`\swarmwrap@fw=\wd\swarmwrap@box`) in the end code after the lrbox closes. The environment now uses lrbox-only capture — content sets its natural width.

For multi-line content (images + captions), users wrap in their own `\begin{minipage}[t]{3cm}...\end{minipage}` inside the swarmwrap environment. This keeps `\captionof` working (minipage is a valid context for it) while giving users full control over content layout.

Left-wrap code was already removed in the previous one-arg v3.0 iteration (llap branch, `\ifswarmwrap@right`). This turn completed the zero-arg migration.

### Changes
- `swarmwrap.sty`: `\newenvironment{swarmwrap}[1]{...}` → `\newenvironment{swarmwrap}{...}`, removed minipage from begin/end, added `\swarmwrap@fw=\wd\swarmwrap@box` in end code
- `test-customwrap.tex`: all 6 tests use `\begin{swarmwrap}` (no arg), each wraps content in explicit `\begin{minipage}[t]{Ncm}` for multi-line content

## 06:00 UTC+8 — Task #119: swarmwrap.sty v3.1 page break fallback

**Files**: `src/themes/swarmwrap.sty`, `src/test-wrapfig/test-customwrap.tex`
**Origin**: QA page break review identified ~160pt+ wasted whitespace when page break triggered.

### Problem

When `\swarmwrapnext` detected insufficient page space for the figure, it inserted `\newpage`, which pushed BOTH the figure AND the wrapped paragraph to the next page. The current page had ~160pt+ of unused whitespace between the vbox/content bottom and the page bottom.

### Approach

Implemented the ALTERNATIVE approach from the task description: when page break triggers, the paragraph fills the remaining space at FULL width (no wrapping), and the figure appears centered at the top of the next page.

Used `\afterpage` package (from CTAN, installed via tlmgr) to defer figure placement. When the page break branch fires:
1. `\afterpage{\swarmwrap@place@centered}` schedules the figure for the next page
2. No parshape is set — text flows at full `\linewidth`
3. The paragraph naturally fills remaining space, overflows to next page
4. On the next page, `\afterpage` fires and places the centered figure before the overflow text

### Changes

## 06:30 UTC+8 — Task #119 improved: parshape transition (right-wrap on next page)

**Files**: `src/themes/swarmwrap.sty`, `src/test-wrapfig/test-customwrap.tex`
**Origin**: Robby requested the better fallback (right-wrap on next page, not centered).

### Approach

Replaced the centered `\afterpage` fallback with a **parshape transition**: when the figure doesn't fit on the current page, a single parshape is computed with N_full full-width lines (to fill remaining space) followed by N_wrap wrapped lines (for the next page). TeX's `\parshape` persists across page breaks within a paragraph, so the paragraph naturally transitions from full-width to wrapped when it breaks to the next page.

The figure overlay is placed on the next page via `\afterpage{\noindent\smash{\rlap{...}}}`. The `\noindent` is critical — it enters horizontal mode at the text margin, ensuring the figure is positioned at the correct horizontal offset (matching normal wrapping).

### Key technical insight

TeX's `\parshape` specification covers ALL lines of a paragraph, including those that fall on subsequent pages after a page break. The line counter does NOT reset at page boundaries. This allows a single parshape to encode a "full-width → wrapped → full-width" transition that spans multiple pages within one paragraph.

### Results

- **pagebreak-variations.tex**: Scenarios F/G/H show correct right-wrapping on the next page with 13.9pt gap. Zero errors, 15 pages.
- **test-customwrap.tex**: 8 pages (was 7 with centered fallback). Extra page is because narrower wrapped lines require more vertical space. The vbox in Test 6 pushes the paragraph continuation far from the figure overlay — a test structure issue.
