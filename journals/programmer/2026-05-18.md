## 06:30 UTC+8 — Task #119 improved: parshape transition (right-wrap on next page)

**Files**: `src/themes/swarmwrap.sty`, `src/test-wrapfig/test-customwrap.tex`
**Origin**: Robby requested the better fallback (right-wrap on next page, not centered).

### Approach

Replaced the centered `\afterpage` fallback with a **parshape transition**: when the figure doesn't fit on the current page, a single parshape is computed with N_full full-width lines (to fill remaining space) followed by N_wrap wrapped lines (for the next page). TeX's `\parshape` persists across page breaks within a paragraph, so the paragraph naturally transitions from full-width to wrapped when it breaks to the next page.

The figure overlay is placed on the next page via `\afterpage{\noindent\smash{\rlap{...}}}`. The `\noindent` is critical — it enters horizontal mode at the text margin, ensuring the figure is positioned at the correct horizontal offset (matching normal wrapping).

### Key technical insight

TeX's `\parshape` specification covers ALL lines of a paragraph, including those that fall on subsequent pages after a page break. The line counter does NOT reset at page boundaries. This allows a single parshape to encode a "full-width → wrapped → full-width" transition that spans multiple pages within one paragraph.

### Results

- **pagebreak-variations.tex**: Scenarios F/G/H show correct right-wrapping on the next page with 13.9pt gap. Zero errors, 15 pages.
- **test-customwrap.tex**: 8 pages (was 7 with centered fallback). Extra page is because narrower wrapped lines require more vertical space. The vbox in Test 6 pushes the paragraph continuation far from the figure overlay — a test structure issue.
- **Fallback reserve**: `\swarmwrap@place@centered` helper preserved. Switch deadline: 2026-05-20.

## 10:15 UTC+8 — Task #123: swarmwrap.sty v3.3 right-wrap page break fallback

**Files**: `src/themes/swarmwrap.sty`, `src/test-wrapfig/test-customwrap.tex`
**Origin**: zoe requested the better fallback option (right-wrap on next page). 3-day lock until 2026-05-20.

### Problem

The v3.2 centered fallback places the figure centered on the next page with zero wrapping. Zoe explicitly requested implementing right-wrapping on the next page instead.

The v3.1 parshape transition approach was abandoned because `\afterpage` figure overlay and parshape narrowed lines were desynchronized.

### Approach — zero-height vbox

Key insight: TeX's `\parshape` persists across page breaks within a paragraph (line counter doesn't reset). So if we set parshape on the current paragraph (even though the figure doesn't fit), the continuation lines on the next page will also be narrowed.

The challenge is placing the figure on the next page WITHOUT resetting the paragraph state. Three approaches were tried and two failed:

1. **`\everypar` approach** (ABANDONED): `\afterpage` sets `\global\everypar` to apply figure + parshape on the first NEW paragraph of the next page. Failed because `\everypar` only fires for NEW paragraphs, not for continuations of paragraphs split by page breaks. User text is typically a single paragraph.

2. **`\afterpage` paragraph approach** (ABANDONED): `\afterpage{\noindent\smash{\rlap{...}}\par}` places the figure on the next page. Failed because creating a paragraph inside `\afterpage` RESETS the parshape for subsequent content. The lipsum continuation lost its narrowing.

3. **Zero-height vbox approach** (SUCCESS): `\afterpage{\vbox to 0pt{\hbox{...figure...}\vss}}` places the figure using a zero-height vbox. Since no paragraph is created, the parshape is NOT reset. TeX doesn't clip vbox overflow — the figure is visible even though the vbox has 0pt height. The figure is positioned at the top-right of the next page, overlapping with the narrowed continuation text.

Critical detail: `\nointerlineskip` must NOT be used after the zero-height vbox inside `\afterpage`. It sets `\prevdepth` to -1000pt, which disrupts the paragraph continuation and creates an extra blank page. Without `\nointerlineskip`, a small amount of interline glue (~3pt) appears between the figure and the first continuation line — acceptable.

### Trade-offs

- **On the current page**: narrowed lines have no figure beside them (figure doesn't fit). This wastes some horizontal space (~99pt per line) but is far better than centered fallback (zero wrapping).
- **On the next page**: figure aligned with narrowed continuation text. Gap ≈ 14pt. After parshape lines exhausted, text returns to full width.

### Results

- test-customwrap.tex: 7 pages, zero errors. Test 6 shows narrowed text on current page (3 lines at w=259.7), figure + narrowed continuation on next page (8 lines, figure at x=391.4, gap=13.9pt).
- test-pagebreak-variations.tex: 15 pages, zero errors. Scenarios F/G/H show correct right-wrapping on figure pages.
- demo-beautiful.tex: 7 pages, no regressions.

## 12:00 UTC+8 — Task #125: swarmwrap.sty v3.4 page-eject fallback

**Files**: `src/themes/swarmwrap.sty`
**Origin**: QA #124 (7/10) — text overlaps figure on 4/8 pages, ghost narrowing on 6/8 pages in v3.3.

### Problem

The v3.3 zero-height `\afterpage` vbox approach had a fundamental flaw: the vbox always placed the figure at the top of the continuation page, but TeX's parshape may have already exhausted its narrow lines on the current page. When most narrow lines fit on the current page, few remain for the continuation — full-width text overlaps the figure.

Specifically, on test-pagebreak-variations.tex (2.5cm figures with caption, fh=135.5pt, N_wrap=10):
- Pages 2, 7, 9, 11: full-width text (x1=476.5) covering figure rectangle (x=391.4-476.5)
- Pages 3, 4, 13, 15: ghost narrowing (narrow text below figure with nothing to wrap)

The root cause is unfixable within the parshape approach: TeX distributes lines between pages based on glue stretch, penalties, and available space — unpredictable at `\swarmwrapnext` time.

### Approach — page eject

Replaced the zero-height vbox with a simple `\newpage` eject. When the figure doesn't fit on the current page:
1. `\newpage` ejects to a fresh page (with full `\textheight` available)
2. Standard wrapping code applies identically to the NORMAL path (parshape + `\smash{\rlap{...}}`)
3. The figure and narrow text are guaranteed to be on the same page

This merges the FALLBACK and NORMAL paths — the only difference is the `\newpage` before wrapping. The code is simpler (fewer branches) and the overlap bug is impossible by construction.

### Trade-offs

- **Current page**: remaining space at the bottom is unused (same as v3.2 centered fallback)
- **Next page**: proper right-wrapping with 14pt gap (better than v3.2's centered figure with zero wrapping)
- **Ghost narrowing**: still present on NORMAL-path continuation pages (parshape persists across page breaks but figure does not). This is cosmetic only — no text overlap.

### Changes

- `swarmwrap.sty` v3.4:
  - FALLBACK branch: replaced zero-height vbox + `\afterpage` with `\newpage`
  - NORMAL and FALLBACK paths now share identical wrapping code (DRY)
  - Removed `\RequirePackage{afterpage}` (no longer needed)
  - Updated header comments to document v3.4 changes and known limitations
  - Bumped `\ProvidesPackage` to v3.4
