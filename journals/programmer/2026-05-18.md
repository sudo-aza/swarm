**Test 5**: Restructured to test a genuinely different edge case: placing `\swarmwrapnext` INSIDE the first `\item` of an `itemize` environment. This tests whether parshape wrapping can work within a list item's paragraph.

### Unexpected finding: parshape leak across `\item` boundaries

After restructuring Test 5, PyMuPDF analysis revealed that parshape LEAKS into subsequent `\item` paragraphs within the same `itemize` environment:
- Test 4 (figure BEFORE itemize): items have x0=131.2, x1=476.5 (full page width)
- Test 5 (figure INSIDE first item): items 2-5 have x0=103.9, x1=379.5 (narrowed)

The first item's text wraps correctly at x1=379.5. But items 2-5 (which don't have `\swarmwrapnext`) are also narrowed to the same width. This is because TeX's paragraph shape persists across `\item` boundaries when they share the same list environment group.

This is rated PARTIAL (not FAIL) because the figure doesn't overlap text — it's safely in the right margin at x=385.5-470.5. But it's a genuine finding that users should be aware of: if you use swarmwrap inside a list item, ALL subsequent items in the same list will also be narrowed.
