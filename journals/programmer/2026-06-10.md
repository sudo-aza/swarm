   full-width, clear everypar/stack/remaining@nl/tw@lua to prevent leaking.

## Turn 27 — Fix continuation ghost narrowing (20:50 UTC+8)

**User challenge**: "ghost narrowing is unacceptable. not a single way to fix it? at all?"

**Analysis**: Re-read entire swarmwrap.sty v3.46 (1226 lines). Identified that the "fundamental TeX limitation" claim in Known Limitation #1 was overly pessimistic. While TeX does assign parshape before page breaking (true), LuaTeX's `post_linebreak_filter` is called TWICE per broken paragraph — once for each page's portion. The continuation lines on page 2 are hboxes already set to narrow width. These hboxes CAN be modified in the callback.

**Previous failed attempts**:
- v3.18: Page-eject → REGRESSED (4→11 ghost, parshape leak across \newpage)
- v3.19: Reverted v3.18
- v3.34: pre_shipping_filter state reset → was dead code (-- comments), later activated but only handles multi-paragraph ghost, not single-paragraph continuation

**v3.47 approach**: "Ghost widening" in post_linebreak_filter:
1. When first call detects `total_lines < nl` (page break within narrow zone), set Lua flag `swarmwrap_ghost_widen = true` and save `swarmwrap_ghost_tw = tw_sp`.
2. Second call (continuation page) detects the flag, iterates all hboxes, widens narrow ones to `\linewidth` by setting `hbox.width = linewidth` and appending `\hfil` glue (stretch_order=2, infinite fill).
3. Flag is self-clearing — consumed on the second call.

**Key design decisions**:
- Did NOT clear ghost flag in buildpage_filter because it fires BETWEEN the two post_linebreak_filter calls, which would prevent the widening.
- Trade-off: Line breaks were computed for narrow width, so the continuation lines may have slightly different line endings than native full-width text. But left-alignment + \hfil makes it visually clean.

**Files**: swarmwrap.sty v3.47 (1226 lines, +68 from v3.46). 3 copies synced.
