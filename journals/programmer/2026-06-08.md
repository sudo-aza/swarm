### Task: Self-task ST-009 — Reduce ghost narrowing via penalty fence (v3.33)

**PROBLEM**: Ghost narrowing (Known Limitation #1) — when a wrapped paragraph
spans a page break, TeX's parshape persists across the boundary but the
figure does not (placed via \smash{\rlap}). Narrow lines on the continuation
page have no figure beside them. 107 pages on the 1000-figure test.

**ROOT CAUSE ANALYSIS**: Two types of ghost narrowing:
- Type A (inter-figure): Narrow text from Figure A extends below Figure A's
  bottom into the gap before Figure B on the SAME page. Analysis tool false
  positives — narrow text is actually wrapping around Figure B.
- Type B (page-break): Paragraph spans page break; narrow lines from the
  parshape leak onto the continuation page with no figure. This is the real
  defect.

Previous attempts:
- v3.18 page-eject: REGRESSED 4→11 ghost on 50-page test (leaked parshape
  state across \newpage). REVERTED in v3.19.
- Boundary penalty (v3.6→v3.29): Penalty inserted after LAST narrow line.
  Prevents break between narrow and full-width text. But TeX can still
  break BETWEEN individual narrow lines, leaking them to next page.

**FIX (v3.33) — Two-part approach:**

1. **Penalty fence**: Insert \penalty between EVERY pair of consecutive
   narrow lines (not just after the last one). In post_linebreak_filter,
   collect all narrow-line nodes, then insert penalty_val between each
   pair. This forces TeX to keep the entire narrow zone on one page or
   push it all to the next — no partial leaks.

2. **Narrow-line space check**: After computing nl, check if
   nl * baselineskip > remaining page space. If so, force DEFERRED.
   This catches edge cases where fh fits but the ceiling computation
   produces more narrow lines than available space.

### Task: #177 — Investigate and fix penalty fence zero effect

**CRITICAL DISCOVERY**: The `post_linebreak_filter` Lua callback was NEVER
DEFINED since v3.17. Verified empirically:

```lua
texio.write_nl("swarmwrap_post_lb = " .. tostring(swarmwrap_post_lb))
-- Output: swarmwrap_post_lb = nil
```

**ROOT CAUSE**: Lua `--` comments inside `\directlua{...}` blocks are FATAL.

TeX's token scanner converts newlines to spaces (catcode 5 → catcode 10)
before passing the string to Lua. In Lua, `--` comments extend to end-of-line
(\n). But TeX already stripped all \n, so `-- comment rest_of_code` becomes
ONE LINE from Lua's perspective, and everything after `--` is consumed.

**Proof of concept**:
```latex
\directlua{
  local x = 1
  texio.write_nl("x="..x)
}
% Output: x=1  ✅

\directlua{
  local x = 1 -- comment
  local y = 2
  texio.write_nl("x="..x.." y="..y)
}
% Output: x=1  (y assignment EATEN by comment)
% Warning: no pages of output  ❌
```

**Impact**: The entire `\directlua{...}` block (76 lines) was dead code.
All features attributed to this callback were actually working via TeX-level
mechanisms alone:

| Feature | Version | Actually Worked? | Why |
|---------|---------|-------------------|-----|
| Remaining-nl tracking | v3.17 | NO (never decremented) | everypar checked remaining@nl > 0 which was set once and never decremented, so stayed positive |
| Cross-page fix | v3.19 | NO | Relied on post_linebreak_filter which was nil |
| Penalty fence | v3.33 | NO | Never inserted any penalties |
| pre_shipping_filter | v3.34 | NO (also had -- comments) | Callback failed to register |
| Emergencystretch reset | v3.15 | NO (leaked everywhere) | Reset code was in dead callback |
| Remaining-height vspace | v3.14 | NO | Computed as 0 always |

**FIX (v3.35)**: Removed ALL `--` Lua comments from both `\directlua{...}`
blocks. Moved documentation to TeX `%` comments outside the blocks. Added
WARNING comment explaining the restriction.
