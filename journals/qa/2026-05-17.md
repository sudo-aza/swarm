**Critical bug found ‚Äî Savebox content lost on group exit (3/10)**:
The `swarmwrap` environment creates a TeX group. Inside it, `\begin{lrbox}{\swarmwrap@box}...\end{lrbox}` fills a `\newsavebox` register with the figure content. Box register assignments in TeX are LOCAL to the current group ‚Äî when `\end{swarmwrap}` closes the group, the savebox reverts to empty. The Programmer used `\xdef` for parshape macros (global, persists), which is why text wrapping works. But `\copy\swarmwrap@box` in `\swarmwrapnext` copies an empty box, so no figure is rendered.

Evidence:
- PyMuPDF: 0 dark pixels in figure areas (x=380-480pt on page 1, x=72-170pt on page 2)
- No `\captionof` text found in extracted PDF text (savebox contents gone)
- Text wrapping measurements are correct (parshape from `\xdef` persists)

**Additional issue**: Test 6 missing `\swarmwrapnext` call after the environment.

**Rating: 3/10 ‚Äî FAIL**

The core functionality (showing a wrapped figure) is broken. Text wrapping works but is pointless without the figure visible. Fix requires making the savebox global: add `\global\setbox\swarmwrap@box=\box\swarmwrap@box` before the group closes.
