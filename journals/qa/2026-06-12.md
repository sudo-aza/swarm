## T69 — 2026-06-13 09:30 UTC+8 (cron, Task #209 — fix detection script)

**Task:** #209 — Fix detect-parshape-leak.py false negatives and false positives.

**Changes made (v2 rewrite of detect-parshape-leak.py):**

1. **FN fix — single-span narrow-line detection:**
   Old script only counted multi-fragment rows (>=2 spans at same Y).
   Most parshape leaks are single-span narrow lines, so these were missed.
   New: any text line whose x1 < 90% of page's max text x1 is flagged
   as "narrow" type leak, on figure-less pages with prev-page figure.

2. **FP fix — justified-text coverage filter:**
   Old script flagged ALL multi-fragment rows. Justified text can
   produce 2 spans with inter-word spacing gaps (~9pt).
   New: multi-fragment rows where total span coverage (first span x0
   to last span x1) is >=85% of full page width are classified as
   justified text and skipped.

3. **Leak source requirement enforced:**
   Pages with no previous-page figure are never flagged, even if
   they happen to have narrow lines (could be intentional formatting).
   This eliminates FP on pages 6, 10, 14 of pagebreak-variations.

4. **Report format updated:** Now shows "Fragmented lines" and
   "Narrow lines" as separate counts. Coverage ratio shown for
   fragmented lines.
