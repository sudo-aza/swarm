#### Task #11: swarmbeauty.sty + demo-beautiful.tex — Rating: 6/10

**What's Good:**
- Well-structured .sty with clear sections and comments
- Good color palette (10 colors, professional look)
- Comprehensive feature set: title page, blocks, code, tables, math theorems, TOC, headers/footers
- Demo tex covers every feature thoroughly
- Engine check at load time (rejects pdflatex)
- `codeblock` environment with minted + tcolorbox is elegant
- Theorem environments with colored left borders look professional

**Issues Found (assigned as Task #16):**

1. **CRITICAL — geometry + KOMA conflict (line 93):** Using `\RequirePackage[margin=2.5cm, a4paper]{geometry}` with `scrartcl` triggers KOMA warnings and can cause layout issues. KOMA has its own typearea system. Fix: use `\usepackage[typearea]{geometry}` or replace with KOMA's `\areaset{...}` commands.

2. **CRITICAL — tocloft + KOMA conflict (line 301):** `tocloft` overrides KOMA's TOC internals via `tocbasic`, causing potential breakage. Fix: use KOMA's own `\setkomafont{tocentry}{...}`, `\RedeclareSectionCommand[tocstyle=...]{section}`, etc.

3. **MODERATE — fancyhdr + KOMA conflict (line 183):** KOMA provides `scrlayer-scrpage` for headers/footers. Using `fancyhdr` alongside KOMA can cause double header lines or conflicts. Fix: replace with `\usepackage[headsepline, footsepline]{scrlayer-scrpage}`.

4. **MODERATE — subcaption + KOMA conflict (line 50):** KOMA has built-in subcaption support. Loading `subcaption` separately may conflict. Fix: use KOMA's `\captionsetup[sub]{...}` instead.

5. **MODERATE — Table rule colors don't work (lines 194-196):** `\swarmtoprule` uses `\color{sbPrimary}\specialrule{...}` but `\color` doesn't color booktabs rules — it leaks into the next cell's text. Fix: use `\arrayrulecolor{sbPrimary}` before the `\specialrule` call.

6. **MODERATE — Title page text overlaps header bar (line 149):** `\vspace*{1.5cm}` places the title text at 1.5cm from top, but the TikZ header bar extends to 3cm. The title renders inside the colored bar. Fix: increase vspace to ~3.5cm or add `\vspace*{3.2cm}`.

7. **MINOR — Redundant color definitions:** `sbDark` (#2C3E50) is identical to `sbPrimary` (#2C3E50). Should be deduplicated or sbDark should be a slightly different shade.

8. **MINOR — Redundant `\providecommand{\insertsubtitle}` (lines 107-110):** KOMA already defines `\insertsubtitle`. The `\providecommand` is a no-op for KOMA classes. Remove to avoid confusion.

#### Task #13: compile.py — Rating: 7/10

**What's Good:**
- Clean argparse interface with `--engine`, `--clean`, `--watch`
- Auto-detects bibliography engine (bibtex vs biber)
- Watch mode with file extension filtering
- Smart TeX Live bin detection (repo-local then system)
- Good error output (last 20 lines on failure)
- Clean aux file list is comprehensive

**Issues Found (assigned as Task #17):**

1. **CRITICAL — No `--shell-escape` support (line 58):** The theme requires `--shell-escape` for minted, but `compile.py` never passes this flag. The default engine is also `pdflatex` but the theme requires `lualatex`. Fix: add a `--shell-escape` flag, and auto-detect minted/tikz usage in the .tex file to enable it automatically.

2. **MODERATE — 3 compilation passes always run (lines 62, 69, 72):** Even without bibliography or cross-references, the script runs 3 passes. This wastes time. Fix: run 1 pass initially, then 2 more only if bib is detected or there are undefined references in the log.

3. **MODERATE — `str | None` type hints (lines 41, 51):** Requires Python 3.10+. While the current VM has 3.12, users on 3.9 will get `TypeError` at import. Fix: use `Optional[str]` from `typing` for broader compatibility.

4. **MINOR — stderr not shown on success:** When compilation succeeds, any warnings on stderr are silently discarded. Fix: print stderr lines containing "Warning" even on success.

#### Task #14: metrics.lua — Rating: 5/10

**What's Good:**
- Uses `callback.register("shipout")` for page counting
- Attempts to measure both wall time and CPU time
- Writes output as JSON
- Defines TeX commands `\metricstart` and `\metricend` for easy use

**Issues Found (assigned as Task #18):**

1. **CRITICAL — `os.time()` for wall time (lines 26, 86-89):** `os.time()` has 1-second resolution. Most LaTeX compilations take 1-10 seconds, so the wall time measurement will be wildly inaccurate (always rounds to 0 or 1). Fix: use `os.clock()` for elapsed time (which it already does for CPU time), or use `os.gettimeofday()` if available, or use the `stop_run`/`start_run` callbacks as the Researcher suggested.

2. **CRITICAL — File inclusion tree never populated (lines 46-63):** `track_input` and `track_input_end` are defined but never registered as hooks. The `orig_input` variable checks for `latex.luatexbase.register_command` but doesn't actually call it. The `inputs` table will always be empty. Fix: use `luatexbase.register_command` or hook into the `input` callback.

3. **MODERATE — JSON serialization is broken for strings (lines 103-104):** Uses `string.format('"%s": "%s"', k, v)` without escaping. If any value contains `"` or `\`, the JSON will be invalid. Fix: add a `json_escape` helper that escapes backslashes, quotes, and control characters.

4. **MODERATE — Table values output as count only (line 106):** `type(v) == "table"` outputs `#v` (length). The `inputs` table structure is lost — only the count appears in JSON. Fix: recursively serialize tables.

5. **MODERATE — Dead counters (lines 39-40):** `font_changes` and `color_changes` are initialized but never incremented anywhere. Either implement the tracking or remove them to avoid confusion.

6. **MINOR — Hardcoded output path (line 98):** `metrics-output.json` is written to the current working directory. Fix: accept an optional path argument or write relative to the main .tex file's directory.

7. **MINOR — `show_warning_message` callback may not exist (line 73):** This callback was added in LuaTeX 1.08+. Should add a pcall or version check.

But `compile.py` looks for `./texlive/bin/x86_64-linux/`, which only matches `setup.sh`'s layout. Running `setup-env.sh` would leave the compile script unable to find LaTeX. Fix: consolidate into one script, or at minimum make both use the same install path.

2. **MODERATE — `setup.sh` uses `sudo` (lines 30-39):** On restricted VMs or environments without passwordless sudo, this fails. `setup-env.sh` correctly avoids sudo for system packages. Fix: make sudo optional with a check and fallback.

3. **MODERATE — `setup.sh` missing `--binary` flag (line 77):** `setup-env.sh` correctly passes `--binary=$PLATFORM` to `install-tl`, but `setup.sh` doesn't. On non-x86_64 systems, this could install the wrong binaries. Fix: add platform detection and `--binary` flag.

4. **MINOR — `TEXLIVE_VERSION="2024"` unused (line 19 of setup.sh):** The variable is defined but never referenced in the install command. Misleading. Fix: either use it or remove it.

5. **MINOR — `sw-push` alias may create empty commits (line 156):** `git commit -m "..."` without `--allow-empty` and without checking for changes will error if there's nothing to commit. Not a disaster but sloppy.
