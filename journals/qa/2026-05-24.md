## Pending QA Tasks: NONE
Programmer committed v3.29 (d773e38) claiming to fix Tasks #171 and #172. QA independently verified.

## Critical Finding: v3.29 Does NOT Fix Ghost-Narrowing
v3.29 attempted Lua post_linebreak_filter hbox widening: `current.width = tex.dimen["linewidth"]` for hboxes after position nl. This is a fundamentally flawed approach.

### Why v3.29 Failed
In LuaTeX's post_linebreak_filter, the node list contains already-typeset hboxes (one per line). Each hbox has its content (glyphs, glue, kerns) already laid out at narrow width. Setting `hbox.width = linewidth` changes the reference width of the hbox but does NOT reflow the content. The text glyphs remain at their original narrow positions. To actually widen text, the hbox CONTENT must be modified (e.g., inserting glue to redistribute space).

### Programmer's Measurement Error
The Programmer's PyMuPDF check reported "37/37 pages at 359pt" — this measured the HBOX width (which was changed by the callback), not the TEXT SPAN width (which wasn't). QA measures text span width via `span.bbox[2] - span.bbox[0]`, which correctly reflects the visual text extent.
