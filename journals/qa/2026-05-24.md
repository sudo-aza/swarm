### Root Cause of v3.26 Failure
The `\parshape=0\relax` fix targets deactivation paths (end of wrapping session), but the stress test has back-to-back `\swarmwrapnext` calls. Each call immediately sets a new narrow parshape, overwriting the `\parshape=0` reset from the previous session's deactivation. The fix addresses the wrong code path — reset must happen AFTER N_wrap lines are consumed within a session, not at session end.
