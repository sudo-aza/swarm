### API assessment

The new API is clean and simple:
```latex
\begin{swarmwrap}%
  \begin{minipage}[t]{3cm}
    \centering
```

Two bugs:
1. **TEXT OVERLAPS FIGURE**: 4/8 pages have full-width text (x1=476.5) covering the figure (x=391.4-476.5). Root cause: N_wrap narrow lines consumed on current page, leaving insufficient narrow lines for continuation page. Parshape resets to full width while figure is still visible.
2. **GHOST NARROWING**: 6/8 pages have narrow text below the figure with nothing to wrap around.

### Root cause analysis

The parshape approach fundamentally cannot guarantee text-figure alignment across page breaks. TeX distributes parshape lines between pages based on available space, which is unpredictable at the time `\swarmwrapnext` sets the parshape. The `\afterpage` zero-height vbox always places the figure at the top of the continuation page, but the narrowed continuation lines may not align with the figure height.

test-customwrap.tex Test 6 works by coincidence: the taller 4cm figure (108pt) results in N_wrap=10 lines covering ~136pt, and only 6 narrow lines fit on page 7, leaving enough for page 8. Shorter 2.5cm figures (with caption, fh=135.5pt but visible rule only 71pt) trigger the mismatch.

### Rating: 7/10 → FAIL

Normal wrapping works perfectly. Page break fallback has significant visual bugs. Created Task #125 for Programmer fix with recommendation to consider centered fallback or Lua callbacks.
