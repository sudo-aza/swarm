**CRITICAL FINDING: Caption-text overlap on 983/1060 pages**

Zoe reported "worse than it was a few hours ago. more overlap" and "the first overlap is on page 1."

### Investigation Steps:

1. **Rendered page 1** of `tests/test-stress-1000.pdf` at 300 DPI (2481x3508 PNG)
2. **Extracted all text spans and figure rectangles** using PyMuPDF
3. **Identified Figure 2** on page 1:
   - Figure box: rect=(391.4, 507.6, 476.5, 592.7) — red!60 fill
   - Figure caption: "Figure 2: Fig 1 (3cmx3cm)" at y=599.8-628.8
4. **Found the overlap**:
   - Narrow wrapping text at y=599.7-615.1, x=117.8-377.5 (correctly wrapping, 259.7pt)
   - Full-width text at y=613.2-628.7, x=117.8-476.5 (**NOT wrapping — goes through caption!**)
   - Full-width text overlaps with caption "(3cmx3cm)" at y=613.4-628.8
   - Overlap area: ~85x15pt = 1314pt²

### Root Cause:
swarmwrap.sty's parshape wrapping zone covers the figure BOX height only, not the full figure+caption height. When a paragraph break occurs within the wrapping zone, the new paragraph starts at full page width, extending through the figure caption text.

### Detection Script Blind Spot:

The existing `detect_overlaps()` function only checked text against figure BOX rectangles (from `get_figures()` which returns filled drawing rectangles). Figure captions are TEXT, not rectangles, so they were invisible to the overlap detection.

This blind spot existed since the detection script was created. ALL previous QA reviews that gave 10/10 PASS ratings (Tasks #235-#266, covering v3.74-v3.85) used this broken detection script.

### Detection Script Fix (Per Rule 10):

Added two new functions to `scripts/detect-layout-issues.py` (v11):
1. `get_figure_caption_zones(page, figures, text_lines)` — finds caption text near each figure box
2. `detect_caption_text_overlap(page_num, caption_zones, text_lines, tolerance)` — detects body text going full-width through caption zones

After fix:
- Page 1: 1 caption-text overlap detected (correct)
- Full 1000-fig: **983 caption-text overlaps** on 1060 pages
