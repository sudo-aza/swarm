## 2026-05-15 04:30 UTC+8 — Task #46: Benchmark swarmmin.sty vs swarmperf.sty

### What I Tested
Compiled `demo-minimal.tex` (swarmmin) and `demo-performance.tex` (swarmperf) with pdfLaTeX, XeLaTeX, and LuaLaTeX. 5 runs x 2 passes each, averaged. Also compiled `demo-beautiful.tex` (swarmbeauty) as reference baseline. Took first-page screenshots. Ran same-document comparison (identical content, only theme changed).

### Benchmark Results (2 passes, 5-run average)

| Theme + Engine | Time | Pages | PDF Size |
|---------------|------|-------|----------|
| swarmmin + pdfLaTeX | **488ms** | 5 | 58KB |
| swarmperf + pdfLaTeX | 765ms | 5 | 77KB |
| swarmmin + XeLaTeX | **730ms** | 5 | — |
| swarmperf + XeLaTeX | 957ms | 5 | — |
| swarmmin + LuaLaTeX | **1637ms** | 5 | — |
| swarmperf + LuaLaTeX | 2173ms | 5 | — |
| swarmbeauty + LuaLaTeX | 6624ms | 7 | 142KB |

Same-document comparison (identical content, pdfLaTeX): swarmmin 464ms vs swarmperf 594ms = **1.3x speedup**.

### Visual Verification
Screenshots saved: `download/swarmmin-page-1.png`, `download/swarmperf-page-1.png`, `download/swarmbeauty-page-1.png`. swarmmin title page is plain text, no decoration. Blocks render as bold-label + normal text.

### Issues Found

1. **CRITICAL BUG: `\sm@loadbooktabs` fails in document body** — Uses `\RequirePackage{booktabs}` inside `\swarmtoprule`/`\swarmmidrule`/`\swarmbottomrule`. These are called inside `tabular` (body). `\RequirePackage` only works in preamble. 6 errors per compile, tables render with no rules. PDF page 4 shows raw tabular without booktabs.

2. **API incompatibility: Theorem environments** — swarmmin uses `\begin{theorem}[name]{label}` (2 args) but swarmperf uses `\begin{theorem}[name]` (1 arg via `\newtheorem`). Cannot swap themes without editing. The `{label}` arg is consumed but unused (dead code).

3. **Documentation inaccurate** — Claims "API-compatible with swarmbeauty (same command/environment names)" but: `\swarmtitlepage` vs `\maketitle`, `\swarmtoprule` vs `\perftoprule`, `\emphtext` only in swarmmin, `\code` different output.

### Rating: 5/10
Speed advantage is real (1.3x over swarmperf, 13.6x over swarmbeauty). But critical booktabs bug means tables don't work, and theorem API incompatibility breaks the swap-themes-without-editing value prop.
