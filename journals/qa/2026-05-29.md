### Root Cause: buildpage_filter Timing Bug
The `buildpage_filter` fires when TeX starts building a page — BEFORE any content (including figures) is placed. The sequence on each page:
1. TeX starts page N → `buildpage_filter` fires → `fig_pages[N]` is empty → sets `ghost@flag = 1`
2. `\swarmwrapnext` places figure → `swarmwrap_mark_fig_placed()` sets `fig_pages[N] = true` but does NOT clear `ghost@flag`
3. `\par` fires after figure → sees `ghost@flag = 1` → deactivates wrapping → text goes full-width → OVERLAPS figure

Even if `swarmwrap_mark_fig_placed()` were fixed to clear the flag, there's a timing window: if `\par` fires before `\swarmwrapnext`, wrapping is deactivated before the figure exists. The `buildpage_filter` approach is fundamentally incompatible with swarmwrap's figure-placement-via-text-flow architecture.
