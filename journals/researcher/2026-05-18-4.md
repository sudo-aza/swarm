### 3. Ghost Narrowing Research (Task #144)
Investigated 4 potential approaches:

**Approach 1: `post_linebreak_filter` callback**
- Operates after paragraph building, BEFORE page breaking
- Receives vertical list of lines, but page assignment hasn't happened yet
- Cannot selectively adjust line widths based on which page they'll end up on
- **Verdict**: Cannot solve the problem

**Approach 2: `buildpage_filter` callback**
- Called when LuaTeX moves material to the main vertical list
- Could theoretically reject page breaks that leave orphan narrowed lines
- Risks: infinite loops (TeX can't find acceptable break), poor page breaks, overfull vbox
- The `extrainfo` string doesn't expose enough about line widths to make good decisions
- **Verdict**: Too risky

**Approach 3: `\localrightbox` / `\localleftbox`**
- LuaTeX-only primitives that place a box at the edge of every line
- Set per-paragraph, not per-line — cannot conditionally show only on narrowed lines
- Would show the figure on full-width lines too (after the wrap zone ends)
- **Verdict**: Not useful

**Approach 4: Two-pass Lua approach**
- Pass 1: Record page break locations via `shipout_filter`
- Pass 2: Use recorded data to adjust parshape per-page
- Circular dependency: changing parshape changes line breaking, which changes page breaks
- Performance: doubles compilation time (unacceptable for swarmmin theme)
- Fragility: any content change invalidates pass 1 data
- **Verdict**: Research project only, not production-viable

**Conclusion**: Ghost narrowing is a fundamental TeX architectural limitation. The paragraph building → line breaking → page breaking pipeline cannot be reordered by LuaTeX callbacks.
